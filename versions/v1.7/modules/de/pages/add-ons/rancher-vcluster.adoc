= {rancher-short-name} vCluster (Experimentell)
:revdate: 2025-12-31
:page-revdate: {revdate}

[NOTE]
====
*rancher-vcluster* ist ein *experimentelles* Add-on. Es ist nicht im ISO enthalten, aber Sie können es aus dem https://github.com/harvester/experimental-addons[`experimental-addons` Repository] herunterladen. Für weitere Informationen zu experimentellen Funktionen siehe xref:introduction/document-conventions.adoc#_feature_labels[Funktionsbezeichnungen].
====

Das *rancher-vcluster* Add-on ermöglicht es Ihnen, {rancher-short-name} als Arbeitslast im zugrunde liegenden {harvester-product-name} Cluster auszuführen und wird mit https://www.vcluster.com/[vCluster] implementiert.

Das Add-on führt ein geschachteltes K3s-Cluster im `rancher-vcluster` Namespace aus und stellt {rancher-short-name} in diesem Cluster bereit.

Während der Installation wird der Ingress für {rancher-short-name} mit dem {harvester-product-name} Cluster synchronisiert, sodass Endbenutzer auf {rancher-short-name} zugreifen können.

== Installieren des Add-ons

Wenn Sie die {harvester-product-name} kubeconfig-Datei verwenden, können Sie das Add-on installieren, indem Sie den folgenden Befehl ausführen:

[,shell]
----
kubectl apply -f https://raw.githubusercontent.com/harvester/experimental-addons/main/rancher-vcluster/rancher-vcluster.yaml
----

== Konfigurieren des Add-ons

Nach der Installation konfigurieren Sie das Add-on über die {harvester-product-name} UI.

. Gehen Sie zu *Erweiterte -> Add-ons*.

. Suchen Sie das *rancher-vcluster* Add-on und wählen Sie dann *⋮ -> Konfiguration bearbeiten* aus.
+
image::rancher-vcluster/VclusterConfig.png[]

. Geben Sie im Feld *Hostname* einen gültigen DNS-Eintrag ein, der auf die {harvester-product-name} VIP zeigt. Dies ist wichtig, da der vcluster Ingress mit dem übergeordneten {harvester-product-name} Cluster synchronisiert wird. Ein gültiger Hostname wird verwendet, um den Ingress-Verkehr zur vcluster-Arbeitslast zu filtern.

. Geben Sie im Feld *Bootstrap-Passwort* das Bootstrap-Passwort für das neue {rancher-short-name} ein, das auf dem vcluster bereitgestellt wird.

Sobald das Add-on bereitgestellt ist, kann es einige Minuten dauern, bis {rancher-short-name} verfügbar ist.

Sie können dann auf {rancher-short-name} über den von Ihnen bereitgestellten Hostnamen-DNS-Eintrag zugreifen.

Siehe xref:integrations/rancher/virtualization-management.adoc[{rancher-short-name} Integration] für weitere Informationen.

[NOTE]
====
Das Add-on wird auf einem `vcluster` Statefulset bereitgestellt, das ein Longhorn PVC mit dem Namen `data-rancher-vcluster-0` verwendet.

Wenn Sie das Add-on deaktivieren, bleibt das PVC im `rancher-vcluster` Namespace. Wenn Sie das Add-on erneut aktivieren, wird das PVC wiederverwendet und der vorherige Zustand von {rancher-short-name} wird wiederhergestellt.

Wenn Sie die Daten löschen möchten, stellen Sie sicher, dass das PVC gelöscht wird.
====
