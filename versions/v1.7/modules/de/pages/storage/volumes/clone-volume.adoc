== Anleitung: Ein Volume klonen

Nachdem Sie ein Volume erstellt haben, können Sie das Volume klonen, indem Sie die folgenden Schritte befolgen:

. Klicken Sie auf die Schaltfläche `⋮` und wählen Sie die Option `Clone` aus.
+
image::volume/clone-volume-1.png[clone-volume-1]

. Auswählen `clone volume data`.
. Konfigurieren Sie `Name` des neuen Volumes und klicken Sie auf `Create`.
. (Optional) Ein geklontes Volume kann mit `Add Existing Volume` zu einer VM hinzugefügt werden.
+
image::volume/clone-volume-2.png[clone-volume-2]

=== Replikat-Wiederherstellung für abgetrennte geklonte {longhorn-product-name} Volumes

Wenn Sie ein abgetrenntes {longhorn-product-name} Volume in einem Multi-Node-Cluster klonen, erstellt {longhorn-product-name} nur _ein Replikat_ anstelle der erwarteten Anzahl (zum Beispiel drei Replikate in einem Drei-Node-Cluster). Der angezeigte Status des Volume-Klons ist `copy-completed-awaiting-healthy` anstelle von `complete`.

{longhorn-product-name} stellt automatisch die verbleibenden Replikate wieder her, nachdem das geklonte Volume an eine virtuelle Maschine angeschlossen wurde. Sie können jedoch den Prozess der Replikat-Wiederherstellung sofort für das abgetrennte Volume starten, indem Sie eine der folgenden Aktionen ausführen:

* In der {longhorn-product-name} UI fügen Sie das Volume manuell hinzu und warten, bis der Prozess der Replikat-Wiederherstellung abgeschlossen ist.

* Aktivieren Sie die {longhorn-product-name} Einstellung https://documentation.suse.com/cloudnative/storage/latest/en/high-availability/offline-replica-rebuilding.html[`offline-replica-rebuilding`], um die automatische Wiederherstellung von Replikaten für abgetrennte Volumes zu ermöglichen.

[NOTE]
====
Dieses Verhalten gilt ausschließlich für abgetrennte Volumes. {longhorn-product-name} führt automatisch die Replikat-Wiederherstellung für Volumes durch, die an einer laufenden virtuellen Maschine angeschlossen sind.
====
