== So bearbeiten Sie eine virtuelle Maschine

Nachdem Sie eine virtuelle Maschine erstellt haben, können Sie Ihre virtuelle Maschine bearbeiten, indem Sie auf die `⋮` Schaltfläche klicken und die `Edit Config` Schaltfläche auswählen.

[NOTE]
====
Neben der Bearbeitung der Beschreibung ist ein Neustart der virtuellen Maschine erforderlich, damit die Konfigurationsänderungen wirksam werden.
====

=== Allgemeine Grundlagen

Im Tab Grundlagen können Sie Ihre angeforderte CPU und den Arbeitsspeicher konfigurieren, ein Neustart der virtuellen Maschine ist erforderlich, damit diese Konfiguration wirksam wird.

SSH-Schlüssel werden in das init-Skript eingefügt, wenn die virtuelle Maschine zum ersten Mal eingeschaltet wird. Damit der modifizierte SSH-Schlüssel nach dem Start der virtuellen Maschine wirksam wird, muss das init-Skript von Ihrem Gastbetriebssystem xref:../troubleshooting/faq.adoc#_how_to_install_the_qemu_guest_agent_of_a_running_vm[neu installiert] werden.

=== Netzwerke

Sie können zusätzliche VLAN-Netzwerke zu Ihren virtuellen Maschineninstanzen nach dem Booten hinzufügen, das `management network` ist optional, wenn Sie das VLAN-Netzwerk konfiguriert haben.

Zusätzliche NICs sind standardmäßig nicht aktiviert, es sei denn, Sie konfigurieren sie manuell im Gastbetriebssystem, z.B. mit https://doc.opensuse.org/documentation/leap/reference/html/book-reference/cha-network.html#sec-network-manconf[wicked für Ihren openSUSE-Server] oder https://ubuntu.com/server/docs/network-configuration[netplan für Ihren Ubuntu-Server].

Wenn die virtuelle Maschine eine Schnittstelle mit dem `mgmt` Netzwerk und eine andere mit einem anderen Netzwerk (z.B. einem VLAN) verbunden hat, kann der Knoten möglicherweise die `mgmt` IP-Adresse der virtuellen Maschine nicht erreichen. Dieses Verbindungsproblem tritt auf, wenn das Gateway des anderen Netzwerks die Standardroute der virtuellen Maschine überschreibt, wodurch für den gesamten eingehenden und ausgehenden Verkehr das VLAN-Netzwerk bevorzugt wird, selbst für den Verkehr, der für das `mgmt` Netzwerk bestimmt ist.

image::vm/edit-vm-networks.png[edit-vm]

Für weitere Details zur Netzwerkimplementierung siehe xref:../networking/vm-network.adoc[Networking].

=== Volumes

Sie können der virtuellen Maschine nach dem Booten zusätzliche Volumes hinzufügen. Sie können die Größe des Volumes auch nach dem Herunterfahren der virtuellen Maschine erweitern: Klicken Sie auf die virtuelle Maschine und gehen Sie zum `Volumes`-Tab, um die Größe des erweiterten Volumes anzupassen. Nach dem Neustart der virtuellen Maschine und dem Warten auf den Abschluss der Größenänderung wird Ihre Festplatte automatisch erweitert.

[IMPORTANT]
====
Wenn Sie externen Speicher verwenden, stellen Sie sicher, dass der Speicheranbieter die Volumenerweiterung unterstützt, bevor Sie die Volumes neu dimensionieren.
====

image::vm/edit-vm-volumes.png[edit-vm]

=== Zugangsdaten

Zugangsdaten ermöglichen es Ihnen, grundlegende Authentifizierung oder SSH-Schlüssel dynamisch zur Laufzeit zu injizieren, wenn Ihr Gastbetriebssystem den QEMU-Gastagenten installiert hat.

Für weitere Informationen siehe xref:./access-vm.adoc#_dynamic_ssh_key_injection_via_qemu_guest_agent[Dynamische SSH-Schlüssel-Injektion über den QEMU-Gastagenten].
