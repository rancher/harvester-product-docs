= Harvester UI Erweiterung
:revdate: 2025-12-30
:page-revdate: {revdate}

{rancher-short-name} 2.10.0 und spätere Versionen integrieren sich mit {harvester-product-name} unter Verwendung der https://github.com/harvester/harvester-ui-extension[Harvester UI Erweiterung], die auf der https://www.npmjs.com/package/@rancher/shell[@rancher/shell] Bibliothek basiert.

[NOTE]
====
Die ältere {harvester-product-name} UI unterstützt weiterhin v1.3.x und v1.4.x. Neue Funktionen in späteren Versionen werden in der Harvester UI Erweiterung implementiert.
====

== Unterstützte Umgebungen

Die Harvester UI Erweiterung ist mit früheren Versionen von {harvester-product-name} kompatibel. Sie verwendet Feature-Flags, um UI-Funktionen anzuzeigen, die der Version des {harvester-product-name} Clusters entsprechen.

Die Versionen des {harvester-product-name} Betriebssystems und der Harvester UI Erweiterung stimmen ab v1.5.0 überein. Um auf neue UI-Funktionen zuzugreifen, installieren Sie die neueste Harvester UI Erweiterung auf Ihrer {rancher-short-name} Instanz.

|===
| Harvester UI Erweiterung Version | Unterstützte {rancher-short-name} Versionen

| v1.0.x
| 2.10.1 und höher

| v1.5.x
| 2.11.0 und höher

| v1.6.x
| 2.12.0 und höher

| v1.7.x
| 2.13.0 und höher
|===

Die Installation der Erweiterung über das Netzwerk ist in Air-Gapped-Umgebungen nicht möglich, daher müssen Sie den in xref:../../installation-setup/airgap.adoc#_harvester_ui_extension_with_rancher_integration[Harvester UI Erweiterung mit {rancher-short-name} Integration] beschriebenen Workaround durchführen.

== Installation auf {rancher-short-name} 2.10

=== Automatisch installieren

Sie können die Harvester UI Erweiterung mit einem einzigen Klick installieren.

. Gehen Sie in der {rancher-short-name} UI zu *Virtualisierungsverwaltung*.
+
image::rancher/auto-install-ui-extension.png[]

. Klicken Sie im Tab *Harvester-Cluster* auf *Installieren*.
+
Lassen Sie etwas Zeit für die Installation der Erweiterung. Der Bildschirm wird automatisch aktualisiert, sobald die Installation abgeschlossen ist.
+
[NOTE]
====
Navigieren Sie während des Installationsprozesses nicht von dem Bildschirm weg.
====

=== Manuelle Installation

Wenn die automatische Installation fehlschlägt, können Sie die URL des Harvester UI Erweiterungs-Repository angeben und anschließend die Erweiterung installieren.

image::rancher/ui-extension-install-failed.png[]

. Gehen Sie im {rancher-short-name} UI zu *lokal -> Apps -> Repositories* und klicken Sie dann auf *Erstellen*.
+
image::rancher/ui-extension-app-repository-setup.png[]

. Konfigurieren Sie die folgenden Einstellungen:
+
* Name: Geben Sie den Namen für das Repository an.
* Ziel: Wählen Sie *das Git-Repository mit Helm-Chart- oder Cluster-Vorlagendefinitionen*.
* Git-Repo-URL: Geben Sie *https://github.com/harvester/harvester-ui-extension*. an.
* Git-Zweig: Geben Sie *gh-pages* an.

. Klicken Sie auf *Erstellen*.

. Gehen Sie zu *Erweiterungen -> Verfügbar*.
+
image::rancher/ui-extension-available-tab.png[]

. Suchen Sie die Erweiterung mit dem Namen *Harvester* und klicken Sie dann auf *Installieren*.

== Aufrüstungen

=== Automatische Upgrades

Der *Aktualisieren* Button erscheint, wann immer eine neue Version der Erweiterung verfügbar ist.

. Gehen Sie in der {rancher-short-name} UI zu *Virtualisierungsverwaltung*.
+
image::rancher/ui-extension-update.png[]

. Klicken Sie auf *Aktualisieren*.
+
Lassen Sie etwas Zeit für das Upgrade der Erweiterung. Der Bildschirm wird automatisch aktualisiert, sobald das Upgrade abgeschlossen ist.

=== Manuelle Upgrades

. Gehen Sie im {rancher-short-name} UI zu *lokal -> Apps -> Repositories*.

. Suchen Sie das Repository mit den folgenden Einstellungen und wählen Sie dann *⋮ -> Aktualisieren* aus.
+
* URL: *https://github.com/harvester/harvester-ui-extension*
* Zweig: *gh-pages*
+
image::upgrade/rancher-2.10.1-repository-page.png[]

. Gehen Sie zu *Erweiterungen -> Installiert*.

. Suchen Sie die Erweiterung mit dem Namen *Harvester* und klicken Sie dann auf *Aktualisieren*.

. Wählen Sie die neue Version aus und klicken Sie dann auf *Aktualisieren*.

image::upgrade/update-harvester-ui-extension-modal.png[]

. Lassen Sie etwas Zeit für das Upgrade der Erweiterung und aktualisieren Sie dann den Bildschirm.

== Nutzungsbeschränkungen

=== {harvester-product-name} v1.2.x und früher werden nicht unterstützt.

Wenn Sie einen Cluster importieren, der {harvester-product-name} v1.2.x oder früher betrieben wird, erscheint eine Fehlermeldung und der Cluster-Link wird deaktiviert.

Der **Verwalten** Button ist in {rancher-short-name} 2.11.0 weiterhin verfügbar, wird jedoch in einer zukünftigen Version deaktiviert. Für weitere Informationen siehe das Problem https://github.com/harvester/harvester/issues/8054[#8054].

[CAUTION]
====
Vermeiden Sie es, den *Verwalten* Button in der {rancher-short-name} Benutzeroberfläche zu verwenden, wenn Ihre Cluster {harvester-product-name} v1.2.x oder früher verwenden. Die Verwaltung von Clustern mit dieser Funktion kann zu unerwartetem Verhalten führen.
====
