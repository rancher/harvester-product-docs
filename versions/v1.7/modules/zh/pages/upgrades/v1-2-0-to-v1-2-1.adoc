== 此版本的重要更改

Harvester `v1.2.1` 修复了在 `v1.2.0` 中发现的已知升级问题，我们建议将 `v1.1.2`、`v1.1.3` 或 `v1.2.0` 集群升级到 `v1.2.1`。在 `v1.2.0` 中发现的已知问题有：

* xref:./v1-1-2-to-v1-2-0.adoc#_9_an_upgrade_is_stuck_in_the_post_draining_state[升级卡在排空后状态]
* xref:./v1-1-2-to-v1-2-0.adoc#_10_an_upgrade_is_stuck_in_the_upgrading_system_service_state_due_to_the_customer_provided_ssl_certificate_without_ip_san_error_in_fleet_agent[由于客户提供的 SSL 证书缺少 IP SAN，在 fleet-agent 中出现错误，导致升级卡在升级系统服务状态]

对于已经升级到 v1.2.0 的集群，请参考 https://github.com/harvester/harvester/releases/tag/v1.2.1[发布说明] 以获取新更改。

== 一般信息

[TIP]
====
在开始升级之前，您可以运行预检查脚本以确保集群处于稳定状态。有关更多详细信息，请访问此 https://github.com/harvester/upgrade-helpers/tree/main/pre-check/v1.1.x[网址] 以获取脚本。
====

一旦有可升级版本，Harvester GUI 仪表板页面将显示升级按钮。有关更多详细信息，请参考 xref:./upgrades.adoc#_start_an_upgrade[开始升级]。

对于隔离环境的升级，请参考 xref:./upgrades.adoc#_prepare_an_air_gapped_upgrade[准备隔离升级]。
