= {rancher-product-name} 集成
:revdate: 2025-09-18
:page-revdate: {revdate}

https://documentation.suse.com/cloudnative/rancher-manager[{rancher-product-name}] 是一个开源的多集群管理平台。{rancher-short-name} 默认集成了 {harvester-product-name}，以集中管理虚拟机和容器。

您可以使用 {rancher-short-name} 的 xref:./virtualization-management.adoc[虚拟化管理] 功能导入和管理多个 {harvester-product-name} 集群，该功能利用了 {rancher-short-name} 的 https://documentation.suse.com/cloudnative/rancher-manager/v2.11/en/rancher-admin/users/authn-and-authz/authn-and-authz.html[认证] 功能和 https://documentation.suse.com/cloudnative/rancher-manager/v2.11/en/rancher-admin/users/authn-and-authz/manage-role-based-access-control-rbac/manage-role-based-access-control-rbac.html[RBAC 控制] 来支持 xref:./virtualization-management.adoc#_multi_tenancy[多租户]。

在执行任何部署步骤之前，请检查 xref:../../installation-setup/requirements.adoc#_network_requirements[网络要求]。

== 部署 {rancher-short-name} 服务器

要将 {rancher-short-name} 与 {harvester-product-name} 一起使用，您必须在单独的服务器上安装 {rancher-short-name}。如果您想尝试集成功能，可以在 {harvester-product-name} 中创建一个虚拟机，并按照 https://documentation.suse.com/cloudnative/rancher-manager/v2.11/en/installation-and-upgrade/quick-start/deploy-rancher/helm-cli.html[Helm CLI 快速入门] 安装 {rancher-short-name} 服务器。

对于生产环境，您可以 https://documentation.suse.com/cloudnative/rancher-manager/v2.11/en/installation-and-upgrade/quick-start/deploy-rancher/deploy-rancher.html[在各种云服务提供商中部署 {rancher-short-name} 并配置 Kubernetes 集群]。

如果您想在本地或与文档中未列出的提供商一起运行 {rancher-short-name}，您可以 https://documentation.suse.com/cloudnative/rancher-manager/v2.11/en/installation-and-upgrade/other-installation-methods/http-proxy/install-rancher.html#_install_the_helm_cli[安装 Helm CLI]，然后使用 Helm 安装 {rancher-short-name}。

[CAUTION]
====
*请勿在生产环境中使用 Docker 安装 {rancher-short-name}*。否则，您的环境可能会损坏，您的集群可能无法恢复。在 Docker 中安装 {rancher-short-name} 仅应用于快速评估和测试目的。
====

== 虚拟化管理

使用 {rancher-short-name} 的虚拟化管理功能，您可以导入和管理您的 {harvester-product-name} 集群。通过点击导入的集群之一，您可以轻松访问和管理一系列 {harvester-product-name} 集群资源，包括主机、虚拟机、镜像、卷等。此外，虚拟化管理功能利用了 {rancher-short-name} 的现有能力，例如与各种认证提供商的认证和多租户支持。

有关深入的见解，请参考 xref:./virtualization-management.adoc[虚拟化管理] 页面。

image::rancher/import-harvester-cluster.png[导入集群]

== 使用 Harvester 节点驱动程序创建 Kubernetes 集群

您可以从 {rancher-short-name} 启动一个 Kubernetes 集群，使用 xref:../../integrations/rancher/node-driver/node-driver.adoc[Harvester 节点驱动程序]。当 {rancher-short-name} 将 Kubernetes 部署到这些节点时，请选择 RKE2。

在由节点驱动程序托管的节点池上安装 Kubernetes 的一个好处是，如果某个节点与集群失去连接，{rancher-short-name} 可以自动创建另一个节点加入集群，以确保节点池的数量符合预期。

Harvester 节点驱动程序默认包含在 {rancher-short-name} 中。有关更多信息，请参见 xref:../../integrations/rancher/node-driver/node-driver.adoc[节点驱动程序]。

image::rancher/harvester-node-driver.png[harvester-node-driver]

== 裸机容器工作负载支持（实验性）

{harvester-product-name} 使您能够直接将容器工作负载部署和管理到底层 {harvester-product-name} 集群。通过此功能，您可以无缝结合虚拟机的强大功能与容器化的灵活性，从而实现更灵活和高效的基础设施设置。

image::rancher/harvester-container-dashboard.png[harvester-container-dashboard]

本指南将引导您启用和使用此实验性功能，突出其能力和最佳实践。

要启用此新功能标志，请按照以下步骤操作：

. 单击汉堡菜单并选择 *全局设置* 选项卡。
. 单击 *功能标志* 并找到新的功能标志 `harvester-baremetal-container-workload`。
. 单击下拉菜单并选择 *激活* 以启用此功能。
. 如果功能状态更改为 *激活*，则表示功能已成功启用。

image::rancher/harvester-baremetal-container-workload-feature.png[harvester-baremetal-container-workload-feature]

=== 主要功能：

* *统一仪表板视图*：启用该功能后，您可以像使用其他标准 Kubernetes 集群一样，探索 {harvester-product-name} 集群的仪表板视图。这个统一的体验简化了从单一用户友好的界面管理和监控虚拟机及容器工作负载的过程。

* *部署自定义工作负载*:此功能允许您将自定义容器工作负载直接部署到裸机 {harvester-product-name} 集群。虽然此功能仍处于实验阶段，但它为优化您的基础设施带来了令人兴奋的可能性。然而，我们建议在不同的名称空间中部署容器和虚拟机工作负载，以确保清晰和分离。

[NOTE]
====
* 监控、日志记录、{rancher-short-name}、KubeVirt 和 Longhorn 等关键系统组件均由 {harvester-product-name} 集群本身管理。您无法升级或修改这些组件。因此，请谨慎行事，避免对这些关键系统组件进行更改。
* 切勿将任何工作负载部署到系统名称空间 `cattle-system`、`harvester-system` 或 `longhorn-system`。将工作负载保持在不同的名称空间中对于维护清晰性和保护系统组件的完整性至关重要。
* 建议在不同的名称空间中部署容器和虚拟机工作负载。
====

== {fleet-product-name} 支持（实验性）

您可以利用 https://documentation.suse.com/cloudnative/continuous-delivery/index.html[{fleet-product-name}] 来管理容器工作负载并使用基于 GitOps 的方法配置 {harvester-product-name}。

[IMPORTANT]
====
必须启用 {rancher-short-name} 功能 `harvester-baremetal-container-workload`。
====

. 在 {rancher-short-name} 用户界面上，转到 *☰ -> 持续交付*。
+
image::rancher/continuous-delivery-overview.png[]

. （可选）在 *集群* 选项卡上，编辑 Fleet 集群配置以添加可用于分组 {harvester-product-name} 集群的标签。
+
在此示例中，添加了标签 `location=private-dc`。
+
image::rancher/fleet-cluster-config.png[]
+
image::rancher/fleet-additional-labels.png[]

. （可选）在 *集群组* 选项卡上，创建一个集群组。
+
在此示例中，集群组 `private-dc-clusters` 是通过与标签键/值对 `location=private-dc` 匹配的集群选择规则创建的。
+
image::rancher/create-cluster-group.png[]

. 在 *Git 储存库* 选项卡上，创建一个名为 `harvester-config` 的 Git 储存库，该储存库指向 https://github.com/harvester/harvester-fleet-examples[harvester-fleet-examples 储存库]，分支定义为 `main`。您必须定义以下路径：
+
* `keypair`
* `vmimage`
* `vmnetwork`
* `cloudinit`
+
image::rancher/gitrepo-definition.png[]

. 单击 *下一步*，然后定义 Git 储存库目标。您可以选择所有集群、单个集群或一组集群。
+
在此示例中，使用名为`private-dc-clusters`的集群组。
+
image::rancher/gitrepo-targets.png[]

. 单击*保存*。资源可能需要几秒钟才能下发到目标集群。
+
image::rancher/gitrepo-synced.png[]
