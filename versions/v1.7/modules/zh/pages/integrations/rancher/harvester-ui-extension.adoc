= Harvester UI 扩展
:revdate: 2025-12-30
:page-revdate: {revdate}

{rancher-short-name} 2.10.0 及更高版本与 {harvester-product-name} 集成，使用 https://github.com/harvester/harvester-ui-extension[Harvester UI 扩展]，该扩展基于 https://www.npmjs.com/package/@rancher/shell[@rancher/shell] 库。

[NOTE]
====
遗留的 {harvester-product-name} UI 继续支持 v1.3.x 和 v1.4.x。后续版本中的新功能在 Harvester UI 扩展中实现。
====

== 支持 Matrix

Harvester UI 扩展与早期版本的 {harvester-product-name} 兼容。它使用功能标志来显示与 {harvester-product-name} 集群版本相对应的 UI 功能。

从 v1.5.0 开始，{harvester-product-name} 操作系统和 Harvester UI 扩展的版本对齐。要访问新 UI 功能，请在您的 {rancher-short-name} 实例上安装最新的 Harvester UI 扩展。

|===
| Harvester UI 扩展版本 | 支持的 {rancher-short-name} 版本

| v1.0.x
| 2.10.1 及更高版本

| v1.5.x
| 2.11.0 及更高版本

| v1.6.x
| 2.12.0 及更高版本

| v1.7.x
| 2.13.0 及更高版本
|===

在隔离环境中无法通过网络安装扩展，因此您必须执行 xref:../../installation-setup/airgap.adoc#_harvester_ui_extension_with_rancher_integration[Harvester UI 扩展与 {rancher-short-name} 集成] 中概述的解决方法。

== 在 {rancher-short-name} 2.10 上安装

=== 自动安装

您可以通过单击一次安装 Harvester UI 扩展。

. 在 {rancher-short-name} UI 上，转到 *虚拟化管理*。
+
image::rancher/auto-install-ui-extension.png[]

. 在 *Harvester Clusters* 选项卡上，单击 *安装*。
+
请等待一段时间以完成扩展的安装。安装完成后，屏幕会自动刷新。
+
[NOTE]
====
在安装过程中，请勿离开此屏幕。
====

=== 手动安装

如果自动安装失败，您可以指定 Harvester UI 扩展的储存库 URL，然后安装该扩展。

image::rancher/ui-extension-install-failed.png[]

. 在 {rancher-short-name} UI 上，转到 *local -> Apps -> Repositories*，然后单击 *创建*。
+
image::rancher/ui-extension-app-repository-setup.png[]

. 配置以下设置：
+
* 名称：指定储存库的名称。
* 目标：选择 *包含 Helm 图表或集群模板定义的 Git 储存库*。
* Git 储存库 URL：指定 *https://github.com/harvester/harvester-ui-extension*.
* Git 分支：指定 *gh-pages*。

. 单击*创建*。

. 转到 *扩展 -> 可用*。
+
image::rancher/ui-extension-available-tab.png[]

. 找到名为 *Harvester* 的扩展，然后单击 *安装*。

== 升级

=== 自动升级

每当有新版本的扩展可用时，*更新* 按钮会出现。

. 在 {rancher-short-name} UI 上，转到 *虚拟化管理*。
+
image::rancher/ui-extension-update.png[]

. 单击 *更新*。
+
请稍等，等待扩展升级完成。升级完成后，屏幕会自动刷新。

=== 手动升级

. 在 {rancher-short-name} UI 上，转到 *local -> Apps -> Repositories*。

. 找到具有以下设置的储存库，然后选择 *⋮ -> 刷新*。
+
* URL：*https://github.com/harvester/harvester-ui-extension*
* 分支：*gh-pages*
+
image::upgrade/rancher-2.10.1-repository-page.png[]

. 前往 *扩展 -> 已安装*。

. 找到名为 *Harvester* 的扩展，然后点击 *更新*。

. 选择新版本，然后点击 *更新*。

image::upgrade/update-harvester-ui-extension-modal.png[]

. 允许一些时间来升级扩展，然后刷新屏幕。

== 局限性

=== {harvester-product-name} v1.2.x 及更早版本不受支持

当您导入运行 {harvester-product-name} v1.2.x 或更早版本的集群时，会出现错误消息，并且集群链接被禁用。

在 {rancher-short-name} 2.11.0 中，**管理** 按钮仍然可用，但在未来的版本中将被禁用。有关更多信息，请参见问题 https://github.com/harvester/harvester/issues/8054[#8054]。

[CAUTION]
====
如果您的集群运行 {harvester-product-name} v1.2.x 或更早版本，请避免在 {rancher-short-name} 界面上使用 *管理* 按钮。使用此功能管理集群可能会导致意外行为。
====
