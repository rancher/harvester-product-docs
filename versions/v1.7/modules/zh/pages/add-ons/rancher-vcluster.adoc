= {rancher-short-name} vCluster（实验性）
:revdate: 2025-12-31
:page-revdate: {revdate}

[NOTE]
====
*rancher-vcluster* 是一个 *实验性* 附加产品。它不包含在 ISO 中，但您可以从 https://github.com/harvester/experimental-addons[`experimental-addons` 储存库] 下载。有关实验性功能的更多信息，请参见 xref:introduction/document-conventions.adoc#_feature_labels[功能标签]。
====

*rancher-vcluster* 附加产品允许您将 {rancher-short-name} 作为工作负载在底层 {harvester-product-name} 集群上运行，并使用 https://www.vcluster.com/[vCluster] 实现。

该附加产品在 `rancher-vcluster` 名称空间中运行一个嵌套的 K3s 集群，并将 {rancher-short-name} 部署到该集群。

在安装过程中，{rancher-short-name} 的入口被同步到 {harvester-product-name} 集群，允许最终用户访问 {rancher-short-name}。

== 安装附加产品

如果您使用 {harvester-product-name} kubeconfig 文件，可以通过运行以下命令安装附加产品：

[,shell]
----
kubectl apply -f https://raw.githubusercontent.com/harvester/experimental-addons/main/rancher-vcluster/rancher-vcluster.yaml
----

== 配置附加产品

安装后，使用 {harvester-product-name} 界面配置附加产品。

. 转到 *高级 -> 附加产品*。

. 找到 *rancher-vcluster* 附加产品，然后选择 *⋮ -> 编辑配置*。
+
image::rancher-vcluster/VclusterConfig.png[]

. 在 *主机名* 字段中，输入指向 {harvester-product-name} VIP 的有效 DNS 记录。这很重要，因为 vcluster 的入口被同步到父 {harvester-product-name} 集群。有效的主机名用于过滤到 vcluster 工作负载的入口流量。

. 在 *启动密码* 字段中，输入在 vcluster 上部署的新 {rancher-short-name} 的启动密码。

一旦附加产品部署，{rancher-short-name} 可能需要几分钟才能可用。

然后，您可以通过您提供的主机名 DNS 记录访问 {rancher-short-name}。

有关更多信息，请参见 xref:integrations/rancher/virtualization-management.adoc[{rancher-short-name} 集成]。

[NOTE]
====
该附加产品部署在一个 `vcluster` 状态集上，使用名为 `data-rancher-vcluster-0` 的 Longhorn PVC。

当您禁用该附加产品时，PVC 仍保留在 `rancher-vcluster` 名称空间中。如果您再次启用该附加产品，PVC 将被重用，{rancher-short-name} 的先前状态将被恢复。

如果您想清除数据，请确保删除 PVC。
====
