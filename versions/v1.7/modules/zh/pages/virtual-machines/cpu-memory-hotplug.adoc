= CPU 和内存热插拔
:revdate: 2025-09-16
:page-revdate: {revdate}

{harvester-product-name} 支持虚拟机的 CPU 和内存热插拔。此功能允许您在不需要重启的情况下增加正在运行的虚拟机的 CPU 和内存资源。

[NOTE]
====
ARM64 架构仅支持内存热插拔。CPU 热插拔操作需要您重启虚拟机。
====

== 在虚拟机创建过程中启用 CPU 和内存热插拔

. 在 *虚拟机：创建* 界面上，指定要分配的 CPU 核心数和内存。
+
image::vm/enable-cpu-and-memory-hotplug.png[启用 CPU 和内存热插拔]

. 选择 *启用 CPU 和内存热插拔*。
+
{harvester-product-name} 会根据 `max-hotplug-ratio` 设置自动填充 *最大 CPU* 和 *最大内存* 字段。这些值表示在运行时可用于虚拟机的最大 CPU 和内存资源。如有必要，您可以更改这些值。

. 完成虚拟机创建的其余步骤。

== 触发 CPU 和内存热插拔操作

. 虚拟机创建后，您可以使用 *编辑 CPU 和内存* 按钮动态添加 CPU 和内存资源。
+
image::vm/edit-cpu-and-memory.png[增加 CPU 和内存资源]

. 点击 *应用* 将 xref:virtual-machines/live-migration#_prerequisites[虚拟机迁移] 到配置了 CPU 和内存数量的节点。

[NOTE]
====
虚拟机必须是 xref:virtual-machines/live-migration.adoc#_live_migratable_virtual_machines[可实时迁移] 的。
====

== `max-hotplug-ratio` 设置

xref:installation-setup/config/settings.adoc#_max_hotplug_ratio[`max-hotplug-ratio`] 是一个全局设置，决定了可用于正在运行的虚拟机的默认最大 CPU 和内存资源。该比例乘以您在创建虚拟机时分配的 CPU 和内存资源的数量。

[sidebar]
运行时可用的最大资源 = （虚拟机创建时分配的资源）x（`max-hotplug-ratio` 值）

您可以指定从 `1` 到 `20` 的任一值。默认值为 `4`。

示例：

`max-hotplug-ratio` 被设置为 `4`。

|===
| 资源 | 分配的数量 | 最大可用数量

| CPU 核心数
| `1`
| `4`

| 内存（Gi）
| `2`
| `8`
|===
