== 如何编辑虚拟机

创建虚拟机后，您可以通过点击`⋮`按钮并选择`Edit Config`按钮来编辑虚拟机。

[NOTE]
====
除了编辑描述外，配置更改生效需要重启虚拟机。
====

=== 基础

在基本选项卡上，您可以配置请求的CPU和内存，配置生效需要重启虚拟机。

SSH 密钥在虚拟机首次启动时注入到 cloud-init 脚本中。为了使修改后的 SSH 密钥在虚拟机启动后生效，需要从您的客户操作系统中 xref:../troubleshooting/faq.adoc#_how_to_install_the_qemu_guest_agent_of_a_running_vm[重新安装] cloud-init 脚本。

image::vm/edit-vm-basics.png[edit-vm]

=== 网络

您可以在虚拟机实例启动后添加额外的VLAN网络，如果您已配置VLAN网络，则`management network`是可选的。

除非您在客户操作系统中手动配置，否则默认情况下不会启用额外的NIC，例如使用 https://doc.opensuse.org/documentation/leap/reference/html/book-reference/cha-network.html#sec-network-manconf[wicked for your OpenSUSE Server]或 https://ubuntu.com/server/docs/network-configuration[netplan for your Ubuntu Server]。

如果虚拟机有一个接口连接到`mgmt`网络，另一个连接到不同的网络（例如VLAN），则节点可能无法访问虚拟机的`mgmt` IP地址。当其他网络的网关覆盖虚拟机的默认路由时，会发生此连接问题，导致路由优先选择VLAN网络进行所有入站和出站流量，即使是目标为`mgmt`网络的流量。

image::vm/edit-vm-networks.png[edit-vm]

有关网络实现的更多详细信息，请参见xref:../networking/vm-network.adoc[网络]。

=== 卷

您可以在虚拟机启动后添加额外的卷。您还可以在关闭虚拟机后扩展卷的大小，点击虚拟机并转到`Volumes`选项卡，编辑扩展卷的大小。在重启虚拟机并等待调整大小完成后，您的磁盘将自动完成扩展。

[IMPORTANT]
====
如果您使用外部存储，请确保存储提供商支持卷扩展，然后再调整卷的大小。
====

image::vm/edit-vm-volumes.png[edit-vm]

=== 访问凭证

访问凭证允许您在客户操作系统安装了 QEMU 客户代理时，在运行时动态注入基本身份验证或 SSH 密钥。

有关更多信息，请参见xref:./access-vm.adoc#_dynamic_ssh_key_injection_via_qemu_guest_agent[通过 QEMU 客户代理动态注入 SSH 密钥]。
