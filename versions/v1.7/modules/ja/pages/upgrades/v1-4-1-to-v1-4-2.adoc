= v1.4.1からv1.4.2にアップグレードする
:revdate: 2025-11-10
:page-revdate: {revdate}

== 一般情報

新しい{harvester-product-name}バージョンが利用可能になると、*ダッシュボード*画面に*アップグレード*ボタンが表示されます。詳細については、xref:./upgrades.adoc#_start_an_upgrade[アップグレードを開始する]を参照してください。

エアギャップ(された)環境については、xref:./upgrades.adoc#_prepare_an_air_gapped_upgrade[エアギャップ(された)アップグレードの準備]を参照してください。

=== {rancher-product-name} v2.10.1でHarvester UI拡張機能を更新する

{harvester-product-name} v1.4.2クラスターを{rancher-short-name} v2.10.1にインポートするには、**v1.0.3**の{rancher-short-name} UI拡張機能を使用する必要があります。

. {rancher-short-name} UIで、*ローカル -> アプリ -> リポジトリ*に移動します。

. *harvester*という名前のリポジトリを見つけて、*⋮ -> 更新*を選択します。
+
このリポジトリには以下のプロパティがあります：
+
* URL: **https://github.com/harvester/harvester-ui-extension**
* ブランチ: **gh-pages**
+
image::upgrade/rancher-2.10.1-repository-page.png[]

. *拡張機能*画面に移動します。

. *Harvester*という名前の拡張機能を見つけて、*更新*をクリックします。

. バージョン*1.0.3*を選択し、*更新*をクリックします。
+
image::upgrade/update-harvester-ui-extension-modal.png[]

. 拡張機能が更新されるまでしばらくお待ちいただき、画面を更新してください。

[IMPORTANT]
====
{rancher-short-name} UIは、拡張機能が更新された後にエラーメッセージを表示します。画面を更新すると、エラーメッセージは消えます。

この問題は{rancher-short-name} v2.10.0およびv2.10.1に存在し、v2.10.2で修正される予定です。
====

関連する問題: https://github.com/harvester/harvester/issues/7234[#7234], https://github.com/rancher/capi-ui-extension/issues/107[#107]

== 仮想マシンバックアップの互換性

外部ストレージを含むバックアップの作成および復元時に、特定の制限に直面する可能性があります。

== 当バージョンの注意事項

=== CPU最大使用率

`backup-target`設定の`refreshIntervalInSeconds`フィールドにより、高いCPU使用率が発生する可能性があります。フィールドが空のままにされるか、`0`に設定されている場合、{harvester-product-name}はバックアップターゲットを常に更新し、高いCPU使用率を引き起こします。

この問題を修正するには、コマンド`kubectl edit setting backup-target`を使用して`refreshIntervalInSeconds`の値を大きな数（例えば、`60`）に更新してください。

例:

[,shell]
----
value: '{"type":"nfs","endpoint":"nfs://longhorn-test-nfs-svc.default:/opt/backupstore", "refreshIntervalInSeconds": 60}'
----

関連する問題: https://github.com/harvester/harvester/issues/7885[#7885]

=== 「Dismiss it」ボタンがクリックされた後、アップグレードが予期せず再起動します。

{rancher-short-name}を使用して{harvester-product-name}をアップグレードすると、{rancher-short-name} UIに「Dismiss it」とラベル付けされたボタンを含むダイアログが表示されます。このボタンをクリックすると、次の問題が発生する可能性があります：

* `status`CRの`harvesterhci.io/v1beta1/upgrade`セクションがクリアされ、アップグレードに関する重要な情報がすべて失われます。
* アップグレードプロセスが予期せず再起動します。

この問題は、v1.0.2、v1.0.3、およびv1.0.4のxref:../integrations/rancher/harvester-ui-extension.adoc#_support_matrix[Harvester UI拡張機能]を使用する{rancher-short-name} v2.10.xに影響します。すべての{harvester-product-name} UIバージョンには影響しません。この問題は、Harvester UI拡張機能 v1.0.5およびv1.5.0で修正されています。

この問題を回避するには、次のいずれかのアクションを実行してください：

* アップグレードには{harvester-product-name} UIを使用してください。{harvester-product-name} UIの「Dismiss it」ボタンをクリックしても、予期しない動作は発生しません。
* {rancher-short-name} UIのボタンをクリックする代わりに、クラスターに対して次のコマンドを実行してください:
+
[,shell]
----
kubectl -n harvester-system label upgrades -l harvesterhci.io/latestUpgrade=true harvesterhci.io/read-message=true
----

関連する問題: https://github.com/harvester/harvester/issues/7791[#7791]

=== 移行可能なRWXボリュームを使用する仮想マシンが予期せず再起動します。

CSIプラグインポッドが再起動されると、移行可能なxref:integrations/rancher/csi-driver.adoc#_rwx_volumes_support[RWXボリューム]を使用する仮想マシンが予期せず再起動します。この問題は {harvester-product-name} v1.4.x、v1.5.0、および v1.5.1 に影響します。

回避策は、アップグレードを開始する前に {longhorn-product-name} UI の設定 https://documentation.suse.com/cloudnative/storage/1.8/en/longhorn-system/settings.html#_automatically_delete_workload_pod_when_the_volume_is_detached_unexpectedly[ボリュームが予期せず切り離されたときにワークロードポッドを自動的に削除する] を無効にすることです。アップグレードが完了したら、再度設定を有効にする必要があります。

この問題は {longhorn-product-name} v1.8.3、v1.9.1、およびそれ以降のバージョンで修正されます。 {harvester-product-name} v1.6.0 には {longhorn-product-name} v1.9.1 が含まれます。

関連する問題: https://github.com/harvester/harvester/issues/8534[#8534] および https://github.com/longhorn/longhorn/issues/11158[#11158]
