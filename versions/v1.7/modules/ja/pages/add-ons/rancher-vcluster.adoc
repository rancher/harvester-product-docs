= {rancher-short-name} vCluster（実験的）
:revdate: 2025-12-31
:page-revdate: {revdate}

[NOTE]
====
*rancher-vcluster* は *実験的* アドオンです。ISOには含まれていませんが、 https://github.com/harvester/experimental-addons[`experimental-addons` リポジトリ] からダウンロードできます。実験的機能に関する詳細は、xref:introduction/document-conventions.adoc#_feature_labels[機能ラベル] を参照してください。
====

*rancher-vcluster* アドオンを使用すると、基盤となる {harvester-product-name} クラスター上で {rancher-short-name} をワークロードとして実行でき、 https://www.vcluster.com/[vCluster] を使用して実装されています。

アドオンは `rancher-vcluster` ネームスペース内でネストされた K3s クラスターを実行し、このクラスターに {rancher-short-name} をデプロイします。

インストール中に、{rancher-short-name} のイングレスが {harvester-product-name} クラスターに同期され、エンドユーザーが {rancher-short-name} にアクセスできるようになります。

== アドオンのインストール

{harvester-product-name} kubeconfig ファイルを使用している場合は、次のコマンドを実行してアドオンをインストールできます：

[,shell]
----
kubectl apply -f https://raw.githubusercontent.com/harvester/experimental-addons/main/rancher-vcluster/rancher-vcluster.yaml
----

== アドオンの設定

インストール後、{harvester-product-name} UI を使用してアドオンを設定します。

. *高度な -> アドオン* に移動します。

. *rancher-vcluster* アドオンを見つけて、*⋮ -> 設定を編集* を選択します。
+
image::rancher-vcluster/VclusterConfig.png[]

. *ホスト名* フィールドに、{harvester-product-name} VIP を指す有効な DNS レコードを入力します。これは、vcluster イングレスが親 {harvester-product-name} クラスターに同期されるため、重要です。有効なホスト名は、vcluster ワークロードへのイングレストラフィックをフィルタリングするために使用されます。

. *ブートストラップパスワード* フィールドに、vcluster にデプロイされた新しい {rancher-short-name} のブートストラップパスワードを入力します。

アドオンがデプロイされると、{rancher-short-name} が利用可能になるまで数分かかることがあります。

その後、提供したホスト名の DNS レコードを介して {rancher-short-name} にアクセスできます。

詳細については、xref:integrations/rancher/virtualization-management.adoc[{rancher-short-name} 統合] を参照してください。

[NOTE]
====
アドオンは、`vcluster` という名前の Longhorn PVC を使用する Statefulset にデプロイされています。

アドオンを無効にすると、PVCは `rancher-vcluster` ネームスペースに残ります。アドオンを再度有効にすると、PVCは再利用され、{rancher-short-name} の以前の状態が復元されます。

データを消去したい場合は、PVCが削除されていることを確認してください。
====
