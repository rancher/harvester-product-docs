= Kit de herramientas del controlador NVIDIA
:revdate: 2025-12-05
:page-revdate: {revdate}

El complemento `nvidia-driver-toolkit` te permite desplegar controladores NVIDIA GRID KVM fuera de banda en tus clústeres {harvester-product-name} existentes.

[NOTE]
====
El kit de herramientas solo incluye la imagen del sistema operativo correcta {harvester-product-name}, utilidades de compilación y encabezados del núcleo que permiten que los controladores NVIDIA se compilen y carguen desde el contenedor. Debes descargar los controladores KVM de NVIDIA utilizando una suscripción válida de NVIDIA. Para obtener orientación sobre cómo identificar el controlador correcto para tu GPU NVIDIA, consulta la https://www.nvidia.com/en-au/drivers/vgpu-software-driver/[documentación de NVIDIA].

Cada nueva versión {harvester-product-name} se lanza con la imagen correcta `nvidia-driver-toolkit` para garantizar que todas las dependencias necesarias para instalar los controladores KVM de NVIDIA vGPU estén disponibles en la imagen.
====

La {harvester-product-name} ISO no incluye la `nvidia-driver-toolkit` imagen del contenedor. Debido a su tamaño, la imagen se extrae de Docker Hub por defecto. Si tienes un entorno aislado, puedes descargar y subir la imagen a tu registro privado. Los campos *Repositorio de Imágenes* y *Etiqueta de Imagen* en la pantalla *nvidia-driver-toolkit* proporcionan información sobre la imagen que debes descargar.

image::advanced/nvidia-driver-toolkit.png[]

Para habilitar el complemento, debes especificar la ubicación HTTP donde se encuentra el archivo del controlador NVIDIA vGPU KVM. También puedes actualizar el repositorio de imágenes y la etiqueta de imagen, si es necesario. Una vez que el complemento está habilitado, se despliega un `nvidia-driver-toolkit` daemonset en el clúster.

Al iniciar el pod, el guion ENTRYPOINT descarga el controlador NVIDIA de la ubicación del controlador especificada. Instala el controlador y carga los controladores del núcleo de Linux.

El complemento `pcidevices-controller` ahora puede aprovechar este complemento para gestionar el ciclo de vida de los dispositivos vGPU en nodos que contienen xref:../hosts/vgpu-support.adoc[dispositivos GPU compatibles].

== Instalando diferentes versiones del controlador NVIDIA

Las versiones de los controladores NVIDIA pueden variar entre los nodos del clúster. Si deseas instalar una versión específica del controlador en un nodo, debes anotar el nodo antes de iniciar el `nvidia-driver-toolkit` complemento.

[,shell]
----
kubectl annotate nodes {node name} sriovgpu.harvesterhci.io/custom-driver=https://[driver location]
----

El complemento instala el controlador especificado al iniciar.

Si se había instalado previamente un controlador de NVIDIA, debes reiniciar el pod para activar de nuevo el proceso de instalación.
