= Operador Kube-OVN (Experimental)
:revdate: 2025-09-15
:page-revdate: {revdate}

[NOTE]
====
**kubeovn-operator** es un *complemento experimental*. Para más información sobre características experimentales, consulte xref:introduction/document-conventions.adoc#_feature_labels[Etiquetas de características].
====

https://github.com/harvester/kubeovn-operator[`kubeovn-operator`] se utiliza para gestionar el ciclo de vida de https://github.com/kubeovn/kube-ovn[Kube-OVN] como un CNI secundario en clústeres subyacentes {harvester-product-name}.

== Habilitando `kubeovn-operator`

Debes habilitar `kubeovn-operator` para desplegar Kube-OVN en un clúster {harvester-product-name} para capacidades avanzadas de SDN como nube privada virtual (VPC) y subredes para cargas de trabajo de máquinas virtuales.

. En la interfaz de usuario de {harvester-product-name}, ve a *Complementos avanzados ->*.

. Selecciona *kubeovn-operator (Experimental)*, y luego selecciona *⋮ -> Habilitar*.
+
image::addons/kubeovn-operator.png[Kube-OVN Operator]

El complemento despliega `kubeovn-operator` y crea el objeto `Configuration` predeterminado llamado `configuration.kubeovn.io`, que utiliza valores predeterminados sensatos específicos de {harvester-product-name} para configurar el CNI de Kube-OVN.

[CAUTION]
====
Este objeto `Configuration` ha sido validado contra instalaciones de {harvester-product-name}. Puedes ajustar el objeto editando el YAML del complemento `kubeovn-operator`, pero ten cuidado al realizar cambios en el campo `configurationSpec`.

Asegúrate de que los bloques CIDR de pod y servicio IPv4 de Kube-OVN no se superpongan con los bloques CIDR de pod y servicio de Harvester.
====

== Deshabilitando `kubeovn-operator`

[IMPORTANT]
====
Asegúrate de que no haya máquinas virtuales utilizando redes de VM respaldadas por componentes SDN de Kube-OVN.

Deshabilitar el complemento kubeovn-operator es un proceso disruptivo.
====

Puedes deshabilitar `kubeovn-operator` utilizando los siguientes comandos:

[,console]
----
kubectl delete configuration kubeovn -n kube-system --wait=false

kubectl delete validatingwebhookconfiguration kube-ovn-webhook --ignore-not-found

kubectl delete ips --all

kubectl delete subnets join ovn-default --ignore-not-found

kubectl delete vpc ovn-cluster --ignore-not-found

# Remove annotations/labels in namespaces and nodes
kubectl annotate node --all ovn.kubernetes.io/cidr-
kubectl annotate node --all ovn.kubernetes.io/gateway-
kubectl annotate node --all ovn.kubernetes.io/ip_address-
kubectl annotate node --all ovn.kubernetes.io/logical_switch-
kubectl annotate node --all ovn.kubernetes.io/mac_address-
kubectl annotate node --all ovn.kubernetes.io/port_name-
kubectl annotate node --all ovn.kubernetes.io/allocated-
kubectl annotate node --all ovn.kubernetes.io/chassis- 
kubectl label node --all kube-ovn/role-
kubectl annotate ns --all ovn.kubernetes.io/cidr-
kubectl annotate ns --all ovn.kubernetes.io/exclude_ips-
kubectl annotate ns --all ovn.kubernetes.io/gateway-
kubectl annotate ns --all ovn.kubernetes.io/logical_switch-
kubectl annotate ns --all ovn.kubernetes.io/private-
kubectl annotate ns --all ovn.kubernetes.io/allow-
kubectl annotate ns --all ovn.kubernetes.io/allocated-

# Remove annotations in all pods of all namespaces
for ns in $(kubectl get ns -o name | awk -F/ '{print $2}'); do
  echo "annotating pods in namespace $ns"
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/cidr-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/gateway-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/ip_address-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/logical_switch-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/mac_address-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/port_name-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/allocated-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/routed-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/vlan_id-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/network_type-
  kubectl annotate pod --all -n $ns ovn.kubernetes.io/provider_network-
done
----

Debes reiniciar cada nodo para completar el proceso de desinstalación. Una vez que los nodos se reinicien, puedes deshabilitar el complemento `kubeovn-operator` desde la interfaz de usuario de Harvester.
