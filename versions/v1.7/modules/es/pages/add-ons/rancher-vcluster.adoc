= {rancher-short-name} vCluster (Experimental)
:revdate: 2025-12-31
:page-revdate: {revdate}

[NOTE]
====
*rancher-vcluster* es un *complemento experimental*. No está incluido en la ISO, pero puedes descargarlo desde el https://github.com/harvester/experimental-addons[`experimental-addons` repositorio]. Para más información sobre características experimentales, consulta xref:introduction/document-conventions.adoc#_feature_labels[Etiquetas de Características].
====

El complemento *rancher-vcluster* te permite ejecutar {rancher-short-name} como una carga de trabajo en el clúster subyacente {harvester-product-name} y se implementa utilizando https://www.vcluster.com/[vCluster].

El complemento ejecuta un clúster K3s anidado en el espacio de nombres `rancher-vcluster` y despliega {rancher-short-name} en este clúster.

Durante la instalación, el ingreso para {rancher-short-name} se sincroniza con el clúster {harvester-product-name}, permitiendo a los usuarios finales acceder a {rancher-short-name}.

== Instalando el complemento

Si estás utilizando el archivo kubeconfig de {harvester-product-name}, puedes instalar el complemento ejecutando el siguiente comando:

[,shell]
----
kubectl apply -f https://raw.githubusercontent.com/harvester/experimental-addons/main/rancher-vcluster/rancher-vcluster.yaml
----

== Configurando el complemento

Después de la instalación, configura el complemento utilizando la interfaz de usuario {harvester-product-name}.

. Ve a *Complementos Avanzados ->*.

. Localiza el complemento *rancher-vcluster*, y luego selecciona *⋮ -> Editar Configuración*.
+
image::rancher-vcluster/VclusterConfig.png[]

. En el campo *Nombre de host*, introduce un registro DNS válido que apunte al VIP {harvester-product-name}. Esto es esencial ya que el ingreso del vcluster se sincroniza con el clúster padre {harvester-product-name}. Se utiliza un nombre de host válido para filtrar el tráfico de ingreso hacia la carga de trabajo del vcluster.

. En el campo *Contraseña de inicio*, introduce la contraseña de inicio para el nuevo {rancher-short-name} desplegado en el vcluster.

Una vez que el complemento esté desplegado, {rancher-short-name} puede tardar unos minutos en estar disponible.

Luego puedes acceder a {rancher-short-name} a través del registro DNS del nombre de host que proporcionaste.

Consulta xref:integrations/rancher/virtualization-management.adoc[{rancher-short-name} Integración] para más información.

[NOTE]
====
El complemento se despliega en un `vcluster` Statefulset que utiliza un PVC de Longhorn llamado `data-rancher-vcluster-0`.

Cuando desactivas el complemento, el PVC permanece en el espacio de nombres `rancher-vcluster`. Si vuelves a habilitar el complemento, el PVC se reutiliza y se restaura el estado anterior de {rancher-short-name}.

Si deseas borrar los datos, asegúrate de que el PVC esté eliminado.
====
