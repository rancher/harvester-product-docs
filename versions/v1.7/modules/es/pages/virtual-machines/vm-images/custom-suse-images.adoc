= Imágenes de Máquina Virtual Personalizadas de SUSE
:revdate: 2026-01-08
:page-revdate: {revdate}

SUSE proporciona https://www.suse.com/download/sles/[SUSE Linux Enterprise (SLE)] y https://get.opensuse.org/leap/[openSUSE Leap] imágenes de máquina virtual adecuadas para su uso en {harvester-product-name}. Estas imágenes se construyen sobre el https://build.opensuse.org/[openSUSE Build Service] (OBS) utilizando la herramienta de construcción de imágenes https://osinside.github.io/kiwi/[Kiwi], y se pueden utilizar inmediatamente después de descargarlas.

En la mayoría de los casos, puedes utilizar las imágenes qcow2 de _Minimal VM Cloud_ porque estas incluyen la herramienta cloud-init necesaria para la configuración automática de la máquina virtual. Otras variantes de imagen requieren que inicies sesión en la consola de la máquina virtual y luego realices la configuración inicial.

[IMPORTANT]
====
Las imágenes _Minimal VM Cloud_ se llamaron _Minimal VM OpenStack Cloud_ en versiones anteriores a SLES 15 SP5 y openSUSE 15.5.
====

== Usando el openSUSE Build Service (OBS)

Puedes crear imágenes personalizadas utilizando las https://build.opensuse.org/image_templates[plantillas de imagen] de OBS de SUSE, que son configuraciones de imagen Kiwi preconfiguradas. Por ejemplo, si deseas utilizar otros paquetes con SLE 15 SP7, puedes crear una imagen utilizando la plantilla _SLE 17 SP5 Minimal_. OBS proporciona una interfaz para añadir paquetes y construye automáticamente la imagen, que puedes descargar y luego subir a {harvester-product-name}. Para más información, consulta la https://openbuildservice.org/help/manuals/obs-user-guide/[Guía del Usuario de OBS].

=== 1. Crea una imagen personalizada basada en una plantilla existente.

. Ve a https://build.opensuse.org/image_templates. Debes iniciar sesión en tu cuenta de openSUSE para acceder a los recursos.
. Selecciona la plantilla que deseas utilizar.
. Especifica un nombre para la imagen y luego selecciona *Crear appliance*.
+
image::advanced/custom-vm-01-select-template.png[]
+
OBS construye automáticamente la imagen. Por defecto, la interfaz muestra la pestaña *Resumen*, que contiene información como el número de paquetes incluidos y el estado de la construcción.
+
image::advanced/custom-vm-02-image-overview.png[]

=== 2. Selecciona perfiles de imagen y añade paquetes.

. Ve a la pestaña *Software*.
. Selecciona los perfiles de imagen que deseas que construya OBS.
+
[IMPORTANT]
====
En la mayoría de los casos, puedes utilizar las imágenes qcow2 de _Minimal VM Cloud_ porque estas incluyen la herramienta cloud-init necesaria para la configuración automática de la máquina virtual. Otras variantes de imagen requieren que inicies sesión en la consola de la máquina virtual y luego realices la configuración inicial.
====
+
image::advanced/custom-vm-03-image-software.png[]

. (Opcional) Añadir y eliminar paquetes.
+
image::advanced/custom-vm-04-image-software-packages.png[]
+
image::advanced/custom-vm-05-image-software-add-package.png[]

=== 3. (Opcional) Cambiar a modo _Ver Paquete_.

El modo _Ver Paquete_ proporciona un control más granular sobre la configuración. Para cambiar, haz clic en el icono _Ver Paquete_ en la barra de navegación.

image::advanced/custom-vm-06-view-package.png[]

La sección *Archivos de origen* de la pestaña *Resumen* muestra todos los archivos que componen tu plantilla de Kiwi. Puedes editar cualquiera de los archivos seleccionando el nombre del archivo correspondiente.

image::advanced/custom-vm-07-view-package-details.png[]

=== 4. (Opcional) Editar el archivo de configuración `Minimal.kiwi`.

Selecciona el nombre del archivo para abrir el editor de texto. La sección `<packages type="image">` lista los paquetes que se van a instalar. Puedes especificar paquetes adicionales para cada perfil de imagen.

Por defecto, el perfil de imagen _Cloud_ (`<package type="image" profiles="Cloud">`) instala el paquete _kernel-default-base_. En el siguiente ejemplo, ese paquete se reemplaza por _kernel-default_, que incluye módulos necesarios para el soporte de iSCSI.

image::advanced/custom-vm-08-edit-kiwi-config.png[]

=== 5. Espera a que OBS termine de construir la imagen.

Una vez completado el proceso, la sección *Resultados de Construcción* en la pestaña *Resumen* muestra el estado _exitoso_.

image::advanced/custom-vm-09-build-succeeded.png[]

La sección *Resultados de Construcción* también contiene un enlace de descarga para la nueva imagen.

image::advanced/custom-vm-10-binaries-link.png[]

image::advanced/custom-vm-11-binaries-download.png[]

=== 6. Habilita la publicación para compartir la imagen.

Para permitir que el público descargue tu imagen personalizada, ve a la pestaña *Repositorios* de tu proyecto OBS y habilita la opción _Publicar_.

image::advanced/custom-vm-12-project-publish-repositories.png[]

Tu imagen se publica en https://download.opensuse.org/ (bajo `repositories/home:/YOUR_USER_NAME:/branches:/SUSE:/Templates:/Images:/`).

image::advanced/custom-vm-13-download.openuse.org.png[]

== Usando la herramienta de línea de comandos de Kiwi

Como alternativa al Servicio de Construcción de openSUSE, puedes crear imágenes localmente utilizando la herramienta de línea de comandos de Kiwi. Para más información sobre la herramienta, consulta https://documentation.suse.com/appliance/kiwi-9/html/kiwi/index.html[Construyendo Aparatos de Sistema Linux con KIWI de Nueva Generación (KIWI NG)].

Para crear imágenes personalizadas, primero debes descargar el archivo `Minimal.kiwi`, y los scripts `config.sh` y `editbootinstall_rpi.sh` del proyecto correspondiente en OBS.
