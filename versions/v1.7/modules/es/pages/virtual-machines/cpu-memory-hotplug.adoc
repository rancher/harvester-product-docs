= Conectar en caliente de CPU y memoria
:revdate: 2025-09-16
:page-revdate: {revdate}

{harvester-product-name} admite conectar en caliente de CPU y memoria para máquinas virtuales. Esta función te permite aumentar los recursos de CPU y memoria de una máquina virtual en ejecución sin necesidad de reiniciar.

[NOTE]
====
La arquitectura ARM64 solo admite conectar en caliente de memoria. Las operaciones de conectar en caliente de CPU requieren que reinicies la máquina virtual.
====

== Habilitar conectar en caliente de CPU y memoria durante la creación de la máquina virtual

. En la pantalla *Crear Máquina Virtual*, especifica el número de núcleos de CPU y la memoria que se asignará.
+
image::vm/enable-cpu-and-memory-hotplug.png[Habilitando conectar en caliente de CPU y memoria]

. Selecciona *Habilitar conectar en caliente de CPU y memoria*.
+
{harvester-product-name} completa automáticamente los campos *Máxima CPU* y *Máxima Memoria* en función de la configuración `max-hotplug-ratio`. Estos valores representan la cantidad máxima de recursos de CPU y memoria que están disponibles para la máquina virtual durante el tiempo de ejecución. Puedes cambiar estos valores, si es necesario.

. Completa el resto de los pasos de creación de la máquina virtual.

== Activar acción de conectar en caliente de CPU y memoria

. Después de crear la máquina virtual, puedes añadir dinámicamente recursos de CPU y memoria utilizando el botón *Editar CPU y Memoria*.
+
image::vm/edit-cpu-and-memory.png[Aumentando recursos de CPU y memoria]

. Haz clic en *Aplicar* para xref:virtual-machines/live-migration#_prerequisites[migrar la máquina virtual] a un nodo con las cantidades de CPU y memoria configuradas.

[NOTE]
====
La máquina virtual debe ser xref:virtual-machines/live-migration.adoc#_live_migratable_virtual_machines[migrable en vivo].
====

== `max-hotplug-ratio` configuración

xref:installation-setup/config/settings.adoc#_max_hotplug_ratio[`max-hotplug-ratio`] es una configuración global que determina la cantidad máxima predeterminada de recursos de CPU y memoria que están disponibles para una máquina virtual en ejecución. Esta relación se multiplica por la cantidad de recursos de CPU y memoria que asignaste al crear la máquina virtual.

[sidebar]
Recurso máximo disponible en tiempo de ejecución = (Recurso asignado durante la creación de la máquina virtual) x (`max-hotplug-ratio` valor)

Puedes especificar un valor entre `1` y `20`. El valor por defecto es `4`.

Ejemplo:

El `max-hotplug-ratio` está configurado en `4`.

|===
| Recurso | Cantidad Asignada | Cantidad Máxima Disponible

| Núcleos de CPU
| `1`
| `4`

| Memoria (Gi)
| `2`
| `8`
|===
