== Cómo editar una máquina virtual

Después de crear una máquina virtual, puedes editar tu máquina virtual haciendo clic en el botón `⋮` y seleccionando el botón `Edit Config`.

[NOTE]
====
Además de editar la descripción, se requiere un reinicio de la máquina virtual para que los cambios de configuración surtan efecto.
====

=== Fundamentos

En la pestaña de básicos, puedes configurar tu CPU y memoria solicitadas, se requiere un reinicio de la máquina virtual para que esta configuración surta efecto.

Las claves SSH se inyectan en el script de cloud-init cuando la máquina virtual se enciende por primera vez. Para que la clave SSH modificada surta efecto después de que la máquina virtual se inicie, el script de cloud-init necesita ser xref:../troubleshooting/faq.adoc#_how_to_install_the_qemu_guest_agent_of_a_running_vm[reinstalado] desde tu sistema operativo invitado.

=== Redes

Puedes añadir redes VLAN adicionales a tus instancias de máquina virtual después de arrancar, el `management network` es opcional si tienes la red VLAN configurada.

Las NIC adicionales no están habilitadas por defecto a menos que las configures manualmente en el sistema operativo invitado, por ejemplo, usando https://doc.opensuse.org/documentation/leap/reference/html/book-reference/cha-network.html#sec-network-manconf[wicked para tu servidor OpenSUSE] o https://ubuntu.com/server/docs/network-configuration[netplan para tu servidor Ubuntu].

Si la máquina virtual tiene una interfaz conectada a la red `mgmt` y otra conectada a una red diferente (como una VLAN), el nodo puede no ser capaz de alcanzar la dirección IP `mgmt` de la máquina virtual. Este problema de conexión ocurre cuando la puerta de enlace de la otra red anula la ruta predeterminada de la máquina virtual, resultando en un enrutamiento que prefiere la red VLAN para todo el tráfico entrante y saliente, incluso el tráfico destinado a la red `mgmt`.

image::vm/edit-vm-networks.png[edit-vm]

Para más detalles sobre la implementación de la red, consulta xref:../networking/vm-network.adoc[Redes].

=== Volúmenes

Puedes añadir volúmenes adicionales a la máquina virtual después de arrancar. También puedes expandir el tamaño del volumen después de apagar la máquina virtual; haz clic en la máquina virtual y ve a la pestaña `Volumes`, editando el tamaño del volumen expandido. Después de reiniciar la máquina virtual y esperar a que se complete el cambio de tamaño, tu disco terminará de expandirse automáticamente.

[IMPORTANT]
====
Si estás utilizando almacenamiento externo, asegúrate de que el proveedor de almacenamiento soporte la expansión de volúmenes antes de redimensionar los volúmenes.
====

=== Credenciales de acceso

Las credenciales de acceso te permiten inyectar autenticación básica o claves SSH de forma dinámica en tiempo de ejecución cuando tu sistema operativo invitado tiene instalado el agente de invitado QEMU.

Para más información, consulta xref:./access-vm.adoc#_dynamic_ssh_key_injection_via_qemu_guest_agent[Inyección dinámica de claves SSH a través del agente de invitado QEMU].
