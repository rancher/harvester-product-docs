= NVIDIA Driver Toolkit
:revdate: 2025-12-05
:page-revdate: {revdate}

Le module complémentaire `nvidia-driver-toolkit` vous permet de déployer des pilotes NVIDIA GRID KVM hors bande sur vos clusters {harvester-product-name} existants.

[NOTE]
====
Le kit d'outils inclut uniquement l'image correcte du système d'exploitation {harvester-product-name}, les utilitaires de construction et les en-têtes du noyau Linux qui permettent aux pilotes NVIDIA d'être compilés et chargés depuis le conteneur. Vous devez télécharger les pilotes NVIDIA KVM en utilisant un abonnement NVIDIA valide. Pour des conseils sur l'identification du bon pilote pour votre GPU NVIDIA, consultez la https://www.nvidia.com/en-au/drivers/vgpu-software-driver/[documentation NVIDIA].

Chaque nouvelle {harvester-product-name} version est publiée avec la bonne `nvidia-driver-toolkit` image pour garantir que toutes les dépendances nécessaires à l'installation des pilotes NVIDIA vGPU KVM sont disponibles dans l'image.
====

L'{harvester-product-name} ISO n'inclut pas l'image du `nvidia-driver-toolkit` conteneur. En raison de sa taille, l'image est tirée par défaut de Docker Hub. Si vous disposez d'un environnement isolé physiquement, vous pouvez télécharger et pousser l'image vers votre registre privé. Les champs *Image Repository* et *Image Tag* sur l'écran *nvidia-driver-toolkit* fournissent des informations sur l'image que vous devez télécharger.

image::advanced/nvidia-driver-toolkit.png[]

Pour activer le module complémentaire, vous devez spécifier l'emplacement HTTP où se trouve le fichier du pilote NVIDIA vGPU KVM. Vous pouvez également mettre à jour le dépôt et le tag d'image, si nécessaire. Une fois le module complémentaire activé, un `nvidia-driver-toolkit` daemonset est déployé sur le cluster.

Au démarrage du pod, le script ENTRYPOINT télécharge le pilote NVIDIA depuis l'emplacement de pilote spécifié. Installez le pilote et chargez les pilotes du noyau Linux.

Le module complémentaire `pcidevices-controller` peut désormais tirer parti de ce module complémentaire pour gérer le cycle de vie des dispositifs vGPU sur les nœuds contenant des GPU pris en charge xref:../hosts/vgpu-support.adoc[dispositifs].

== Installation de différentes versions de pilotes NVIDIA

Les versions des pilotes NVIDIA peuvent varier d'un nœud de cluster à l'autre. Si vous souhaitez installer une version spécifique du pilote sur un nœud, vous devez annoter le nœud avant de démarrer le `nvidia-driver-toolkit` module complémentaire.

[,shell]
----
kubectl annotate nodes {node name} sriovgpu.harvesterhci.io/custom-driver=https://[driver location]
----

Le module complémentaire installe le pilote spécifié lors du démarrage.

Si un pilote NVIDIA a été précédemment installé, vous devez redémarrer le pod pour relancer le processus d'installation.
