= {rancher-short-name} vCluster (Expérimental)
:revdate: 2025-12-31
:page-revdate: {revdate}

[NOTE]
====
*rancher-vcluster* est un *produit complémentaire expérimental*. Il n'est pas inclus dans l'ISO, mais vous pouvez le télécharger depuis le https://github.com/harvester/experimental-addons[`experimental-addons` dépôt]. Pour plus d'informations sur les fonctionnalités expérimentales, consultez xref:introduction/document-conventions.adoc#_feature_labels[Étiquettes de fonctionnalités].
====

Le produit complémentaire *rancher-vcluster* vous permet d'exécuter {rancher-short-name} en tant que charge de travail sur le cluster {harvester-product-name} sous-jacent et est implémenté en utilisant https://www.vcluster.com/[vCluster].

Le produit complémentaire exécute un cluster K3s imbriqué dans l'espace de noms `rancher-vcluster` et déploie {rancher-short-name} sur ce cluster.

Pendant l'installation, l'ingress pour {rancher-short-name} est synchronisé avec le cluster {harvester-product-name}, permettant aux utilisateurs finaux d'accéder à {rancher-short-name}.

== Installation du produit complémentaire

Si vous utilisez le fichier kubeconfig {harvester-product-name}, vous pouvez installer le produit complémentaire en exécutant la commande suivante{nbsp}:

[,shell]
----
kubectl apply -f https://raw.githubusercontent.com/harvester/experimental-addons/main/rancher-vcluster/rancher-vcluster.yaml
----

== Configuration du produit complémentaire

Après l'installation, configurez le produit complémentaire en utilisant l'interface utilisateur {harvester-product-name}.

. Allez à *Add-ons avancés ->*.

. Localisez le produit complémentaire *rancher-vcluster*, puis sélectionnez *⋮ -> Modifier la configuration*.
+
image::rancher-vcluster/VclusterConfig.png[]

. Dans le champ *Nom d'hôte*, entrez un enregistrement DNS valide pointant vers le VIP {harvester-product-name}. C'est essentiel car l'ingress du vcluster est synchronisé avec le cluster parent {harvester-product-name}. Un nom d'hôte valide est utilisé pour filtrer le trafic d'ingress vers la charge de travail du vcluster.

. Dans le champ *Mot de passe de démarrage*, entrez le mot de passe de démarrage pour le nouveau {rancher-short-name} déployé sur le vcluster.

Une fois le produit complémentaire déployé, {rancher-short-name} peut prendre quelques minutes pour devenir disponible.

Vous pouvez ensuite accéder à {rancher-short-name} via l'enregistrement DNS du nom d'hôte que vous avez fourni.

Voir xref:integrations/rancher/virtualization-management.adoc[{rancher-short-name} Intégration] pour plus d'informations.

[NOTE]
====
Le produit complémentaire est déployé sur un `vcluster` Statefulset qui utilise un PVC Longhorn nommé `data-rancher-vcluster-0`.

Lorsque vous désactivez le produit complémentaire, le PVC reste dans l'espace de noms `rancher-vcluster`. Si vous réactivez le produit complémentaire, le PVC est réutilisé et l'état précédent de {rancher-short-name} est restauré.

Si vous souhaitez effacer les données, assurez-vous que le PVC est supprimé.
====
