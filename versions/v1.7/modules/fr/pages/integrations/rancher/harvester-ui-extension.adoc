= Extension de l'interface utilisateur Harvester
:revdate: 2025-12-30
:page-revdate: {revdate}

{rancher-short-name} Les versions 2.10.0 et ultérieures s'intègrent avec {harvester-product-name} en utilisant la https://github.com/harvester/harvester-ui-extension[extension de l'interface utilisateur Harvester], qui est construite sur la bibliothèque https://www.npmjs.com/package/@rancher/shell[@rancher/shell].

[NOTE]
====
L'interface utilisateur héritée {harvester-product-name} continue de prendre en charge les versions v1.3.x et v1.4.x. De nouvelles fonctionnalités dans les versions ultérieures sont mises en œuvre dans l'extension de l'interface utilisateur Harvester.
====

== Matrice de support

L'extension de l'interface utilisateur Harvester est compatible avec les versions antérieures de {harvester-product-name}. Elle utilise des indicateurs de fonctionnalités pour afficher les fonctionnalités de l'interface utilisateur qui correspondent à la version du cluster {harvester-product-name}.

Les versions du système d'exploitation {harvester-product-name} et de l'extension de l'interface utilisateur Harvester s'alignent à partir de v1.5.0. Pour accéder aux nouvelles fonctionnalités de l'interface utilisateur, installez la dernière extension de l'interface utilisateur Harvester sur votre instance {rancher-short-name}.

|===
| Version de l'extension de l'interface utilisateur Harvester | Versions prises en charge de {rancher-short-name}

| v1.0.x
| 2.10.1 et ultérieures

| v1.5.x
| 2.11.0 et ultérieures

| v1.6.x
| 2.12.0 et ultérieures

| v1.7.x
| 2.13.0 et ultérieures
|===

L'installation de l'extension via le réseau n'est pas possible dans des environnements isolés physiquement, vous devez donc effectuer la solution de contournement décrite dans xref:../../installation-setup/airgap.adoc#_harvester_ui_extension_with_rancher_integration[extension de l'interface utilisateur Harvester avec {rancher-short-name} intégration].

== Installation sur {rancher-short-name} 2.10

=== Installation automatique

Vous pouvez installer l'extension de l'interface utilisateur Harvester d'un simple clic.

. Dans l'interface utilisateur {rancher-short-name}, allez à *Gestion de la virtualisation*.
+
image::rancher/auto-install-ui-extension.png[]

. Dans l'onglet *Harvester Clusters*, cliquez sur *Installer*.
+
Laissez un certain temps pour que l'extension soit installée. L'écran est automatiquement actualisé une fois l'installation terminée.
+
[NOTE]
====
Ne quittez pas l'écran pendant le processus d'installation.
====

=== Installation manuelle

Si l'installation automatique échoue, vous pouvez spécifier l'URL du dépôt de l'extension de l'interface utilisateur Harvester, puis installer l'extension.

image::rancher/ui-extension-install-failed.png[]

. Sur l'interface {rancher-short-name}, allez à *local -> Apps -> Dépôts*, puis cliquez sur *Créer*.
+
image::rancher/ui-extension-app-repository-setup.png[]

. Configurez les paramètres suivants :
+
* Nom : Indiquez le nom du référentiel.
* Cible : Sélectionnez *le dépôt Git contenant les définitions de chartes Helm ou de modèles de cluster*.
* URL du dépôt Git{nbsp}: Indiquez *https://github.com/harvester/harvester-ui-extension*.
* Branche Git{nbsp}: Indiquez *gh-pages*.

. Cliquez sur *Create*.

. Allez à *Extensions -> Disponibles*.
+
image::rancher/ui-extension-available-tab.png[]

. Localisez l'extension nommée *Harvester*, puis cliquez sur *Installer*.

== Mises à niveau

=== Mises à niveau automatiques

Le bouton *Mettre à jour* apparaît chaque fois qu'une nouvelle version de l'extension est disponible.

. Dans l'interface utilisateur {rancher-short-name}, allez à *Gestion de la virtualisation*.
+
image::rancher/ui-extension-update.png[]

. Cliquez sur *Mettre à jour*.
+
Laissez un certain temps pour que l'extension soit mise à niveau. L'écran est automatiquement actualisé une fois la mise à niveau terminée.

=== Mises à niveau manuelles

. Sur l'interface {rancher-short-name}, allez à *local -> Apps -> Dépôts*.

. Localisez le dépôt avec les paramètres suivants, puis sélectionnez *⋮ -> Actualiser*.
+
* URL{nbsp}: *https://github.com/harvester/harvester-ui-extension*
* Branche{nbsp}: *gh-pages*
+
image::upgrade/rancher-2.10.1-repository-page.png[]

. Allez à *Extensions -> Installées*.

. Localisez l'extension nommée *Harvester*, puis cliquez sur *Mettre à jour*.

. Sélectionnez la nouvelle version, puis cliquez sur *Mettre à jour*.

image::upgrade/update-harvester-ui-extension-modal.png[]

. Laissez un certain temps pour que l'extension soit mise à niveau, puis actualisez l'écran.

== limites

=== {harvester-product-name} v1.2.x et antérieurs ne sont pas pris en charge

Lorsque vous importez un cluster qui exécute {harvester-product-name} v1.2.x ou antérieurs, un message d'erreur apparaît et le lien du cluster est désactivé.

Le bouton **Gérer** est toujours disponible dans {rancher-short-name} 2.11.0, mais il sera désactivé dans une future version. Pour plus d'informations, voir le problème https://github.com/harvester/harvester/issues/8054[#8054].

[CAUTION]
====
Évitez d'utiliser le bouton *Gérer* sur l'interface {rancher-short-name} si vos clusters exécutent {harvester-product-name} v1.2.x ou antérieurs. La gestion des clusters à l'aide de cette fonctionnalité peut entraîner un comportement inattendu.
====
