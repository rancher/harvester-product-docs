= Installer uniquement des binaires
:revdate: 2025-05-09
:page-revdate: {revdate}

Vous pouvez installer et configurer uniquement des binaires, ce qui est idéal pour les cas d'utilisation dans le cloud et en périphérie.

image::install/choose-installation-mode.png[choose-installation-mode.png]

== Arrière-plan

Actuellement, lorsqu'un nouveau nœud est lancé, il doit être le premier nœud du cluster ou rejoindre un cluster existant.
Ces deux modes sont utiles lorsque vous en savez déjà suffisamment sur l'environnement pour installer le nœud.
Cependant, pour des cas d'utilisation tels que les fournisseurs de cloud sur matériel sans système d'exploitation et la périphérie, ces modes d'installation chargent le système d'exploitation et le contenu sur le nœud sans vous permettre de configurer le réseau. De plus, la configuration K8s et réseau ne sera pas appliquée.

Si vous choisissez `Install Harvester binaries only`, vous devrez effectuer une configuration supplémentaire après le premier démarrage{nbsp}:

* Option Créer/Rejoindre
* Détails de l'interface réseau de gestion
* Token de cluster
* Mot de passe du nœud

Ensuite, l'installateur appliquera la configuration de point de terminaison et démarrera {harvester-product-name}. Aucun redémarrage supplémentaire ne sera nécessaire.

== Mode disque en streaming

SUSE a publié un artefact d'image brute pour pré-installé {harvester-product-name}. L'installateur permet désormais de diffuser une image pré-installée directement sur le disque pour mieux s'intégrer aux fournisseurs de cloud.

Sur `Equinix Metal`, vous pouvez utiliser les arguments de noyau Linux suivants pour utiliser le mode de streaming{nbsp}:

----
ip=dhcp net.ifnames=1 rd.cos.disable rd.noverifyssl root=live:http://${artifactEndpoint}/harvester-v1.2.0-rootfs-amd64.squashfs harvester.install.automatic=true harvester.scheme_version=1 harvester.install.device=/dev/vda  harvester.os.password=password harvester.install.raw_disk_image_path=http://${artifactEndpoint}/harvester-v1.2.0-amd64.raw harvester.install.mode=install console=tty1 harvester.install.tty=tty1 harvester.install.config_url=https://metadata.platformequinix.com/userdata harvester.install.management_interface.interfaces="name:enp1s0" harvester.install.management_interface.method=dhcp harvester.install.management_interface.bond_options.mode=balance-tlb harvester.install.management_interface.bond_options.miimon=100
----

[NOTE]
====
Lors de la diffusion sur le disque, il est recommandé d'héberger l'artefact de disque brut plus près des cibles, car l'artefact de disque brut fait presque 16 Go.
====
