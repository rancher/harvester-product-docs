== Procedimentos para clonar um volume

Após criar um volume, você pode clonar o volume seguindo os passos abaixo:

. Clique no botão `⋮` e selecione a opção `Clone`.
+
image::volume/clone-volume-1.png[clone-volume-1]

. Selecione `clone volume data`.
. Configure o `Name` do novo volume e clique em `Create`.
. (Opcional) Um volume clonado pode ser adicionado a uma VM usando `Add Existing Volume`.
+
image::volume/clone-volume-2.png[clone-volume-2]

=== Reconstrução de réplicas para volumes clonados {longhorn-product-name} desconectados

Quando você clona um volume {longhorn-product-name}desconectado em um cluster de múltiplos nós, {longhorn-product-name} cria apenas _uma réplica_ em vez do número esperado (por exemplo, três réplicas em um cluster de três nós). O status do clone do volume exibido é `copy-completed-awaiting-healthy` em vez de `complete`.

{longhorn-product-name} reconstrói automaticamente as réplicas restantes após o volume clonado ser anexado a uma máquina virtual. No entanto, você pode iniciar imediatamente o processo de reconstrução de réplicas para o volume desconectado realizando uma das seguintes ações:

* Na interface do {longhorn-product-name}, anexe manualmente o volume e aguarde a conclusão do processo de reconstrução da réplica.

* Ative a configuração {longhorn-product-name} https://documentation.suse.com/cloudnative/storage/latest/en/high-availability/offline-replica-rebuilding.html[`offline-replica-rebuilding`] para permitir a reconstrução automática de réplicas para volumes desconectados.

[NOTE]
====
Esse comportamento se aplica exclusivamente a volumes desconectados. {longhorn-product-name} realiza automaticamente a reconstrução de réplicas para volumes que estão anexados a uma máquina virtual em execução.
====
