= Complementos
:revdate: 2025-12-30
:page-revdate: {revdate}

{harvester-product-name} oferece recursos opcionais como complementos para manter uma instalação mínima. Esse design permite que você ative ou desative funcionalidades com base no seu caso de uso e requisitos. Você pode personalizar as configurações de cada complemento, embora o nível de personalização dependa do software subjacente.

Os seguintes complementos estão disponíveis nesta versão:

* xref:add-ons/lvm-local-storage.adoc[csi-driver-lvm] (Experimental)
* xref:add-ons/kubeovn-operator.adoc[kubeovn-operator] (Experimental)
* xref:add-ons/nvidia-driver-toolkit.adoc[nvidia-driver-toolkit]
* xref:add-ons/pcidevices-controller.adoc[pcidevices-controller]
* xref:observability/logging.adoc[rancher-logging]
* xref:observability/monitoring.adoc[rancher-monitoring]
* xref:add-ons/rancher-vcluster.adoc[rancher-vcluster] (Experimental)
* xref:add-ons/harvester-seeder.adoc[seeder] (Experimental)
* xref:add-ons/vm-dhcp-controller.adoc[vm-dhcp-controller] (Experimental)
* xref:add-ons/vm-import-controller.adoc[vm-import-controller]

Você pode ativar e desativar complementos na tela *Complementos* da interface {harvester-product-name}.

|===
| Operação | Ação | Novo Estado

| Ativando um complemento
| Selecione o complemento e, em seguida, selecione *⋮ -> Ativar*.
| **DeploySuccessful**

| Desativando um complemento
| Selecione o complemento e, em seguida, selecione *⋮ -> Desativar*.
| *Desabilitado*
|===

[NOTE]
====
{harvester-product-name} armazena os dados de configuração dos complementos desativados, permitindo a reutilização imediata ao reativá-los.
====
