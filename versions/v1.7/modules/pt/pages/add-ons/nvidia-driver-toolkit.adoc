= Kit de Ferramentas do Driver NVIDIA
:revdate: 2025-12-05
:page-revdate: {revdate}

O `nvidia-driver-toolkit` complemento permite que você implante drivers NVIDIA GRID KVM fora de banda em seus clusters {harvester-product-name} existentes.

[NOTE]
====
O kit de ferramentas inclui apenas a imagem do sistema operacional correta {harvester-product-name}, utilitários de compilação e cabeçalhos de kernel que permitem que os drivers NVIDIA sejam compilados e carregados a partir do contêiner. Você deve baixar os drivers KVM da NVIDIA usando uma assinatura NVIDIA válida. Para orientações sobre como identificar o driver correto para sua GPU NVIDIA, consulte a https://www.nvidia.com/en-au/drivers/vgpu-software-driver/[documentação da NVIDIA].

Cada nova versão {harvester-product-name} é lançada com a imagem correta `nvidia-driver-toolkit` para garantir que todas as dependências necessárias para instalar os drivers KVM vGPU da NVIDIA estejam disponíveis na imagem.
====

A ISO {harvester-product-name} não inclui a imagem do contêiner `nvidia-driver-toolkit`. Devido ao seu tamanho, a imagem é puxada do Docker Hub por padrão. Se você tiver um ambiente air-gapped, pode baixar e enviar a imagem para seu registro privado. Os campos *Repositório de Imagens* e *Tag da Imagem* na tela *nvidia-driver-toolkit* fornecem informações sobre a imagem que você deve baixar.

image::advanced/nvidia-driver-toolkit.png[]

Para habilitar o complemento, você deve especificar o local HTTP onde o arquivo do driver KVM vGPU da NVIDIA está localizado. Você também pode atualizar o repositório de imagens e a tag da imagem, se necessário. Uma vez que o complemento esteja habilitado, um `nvidia-driver-toolkit` daemonset é implantado no cluster.

Na inicialização do pod, o script ENTRYPOINT baixa o driver NVIDIA do local de driver especificado. Instale o driver e carregue os drivers do kernel.

O complemento `pcidevices-controller` agora pode aproveitar este complemento para gerenciar o ciclo de vida dos dispositivos vGPU em nós que contêm xref:../hosts/vgpu-support.adoc[dispositivos GPU suportados].

== Instalando diferentes versões do driver NVIDIA

As versões dos drivers NVIDIA podem variar entre os nós do cluster. Se você quiser instalar uma versão específica do driver em um nó, deve anotar o nó antes de iniciar o `nvidia-driver-toolkit` complemento.

[,shell]
----
kubectl annotate nodes {node name} sriovgpu.harvesterhci.io/custom-driver=https://[driver location]
----

O complemento instala o driver especificado ao iniciar.

Se um driver da NVIDIA foi instalado anteriormente, você deve reiniciar o pod para acionar o processo de instalação novamente.
