= {rancher-short-name} vCluster (Experimental)
:revdate: 2025-12-31
:page-revdate: {revdate}

[NOTE]
====
*rancher-vcluster* é um *experimental* complemento. Não está incluído na ISO, mas você pode baixá-lo do https://github.com/harvester/experimental-addons[`experimental-addons` repositório]. Para mais informações sobre recursos experimentais, veja xref:introduction/document-conventions.adoc#_feature_labels[Rótulos de Recursos].
====

O complemento *rancher-vcluster* permite que você execute {rancher-short-name} como uma carga de trabalho no cluster {harvester-product-name} subjacente e é implementado usando https://www.vcluster.com/[vCluster].

O complemento executa um cluster K3s aninhado no namespace `rancher-vcluster` e implanta {rancher-short-name} neste cluster.

Durante a instalação, o ingress para {rancher-short-name} é sincronizado com o cluster {harvester-product-name}, permitindo que os usuários finais acessem {rancher-short-name}.

== Instalando o complemento

Se você estiver usando o arquivo kubeconfig {harvester-product-name}, pode instalar o complemento executando o seguinte comando:

[,shell]
----
kubectl apply -f https://raw.githubusercontent.com/harvester/experimental-addons/main/rancher-vcluster/rancher-vcluster.yaml
----

== Configurando o complemento

Após a instalação, configure o complemento usando a interface {harvester-product-name}.

. Vá para *Complementos Avançados ->*.

. Localize o complemento *rancher-vcluster* e, em seguida, selecione *⋮ -> Editar Configuração*.
+
image::rancher-vcluster/VclusterConfig.png[]

. No campo *Hostname*, insira um registro DNS válido apontando para o VIP {harvester-product-name}. Isso é essencial, pois o ingress do vcluster é sincronizado com o cluster pai {harvester-product-name}. Um nome de host válido é usado para filtrar o tráfego de ingress para a carga de trabalho do vcluster.

. No campo *Senha de Inicialização*, insira a senha de inicialização para o novo {rancher-short-name} implantado no vcluster.

Uma vez que o complemento esteja implantado, {rancher-short-name} pode levar alguns minutos para ficar disponível.

Você pode então acessar {rancher-short-name} através do registro DNS que você forneceu.

Veja xref:integrations/rancher/virtualization-management.adoc[{rancher-short-name} Integração] para mais informações.

[NOTE]
====
O complemento é implantado em um `vcluster` Statefulset que utiliza um PVC Longhorn chamado `data-rancher-vcluster-0`.

Quando você desativa o complemento, o PVC permanece no namespace `rancher-vcluster`. Se você habilitar o complemento novamente, o PVC é reutilizado e o estado anterior de {rancher-short-name} é restaurado.

Se você deseja apagar os dados, certifique-se de que o PVC seja excluído.
====
