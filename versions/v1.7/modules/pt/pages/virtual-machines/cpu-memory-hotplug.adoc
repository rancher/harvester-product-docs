= Hotplug de CPU e Memória
:revdate: 2025-09-16
:page-revdate: {revdate}

{harvester-product-name} suporta hotplug de CPU e memória para máquinas virtuais. Esse recurso permite aumentar os recursos de CPU e memória de uma máquina virtual em execução sem precisar reiniciar.

[NOTE]
====
A arquitetura ARM64 suporta apenas hotplug de memória. Operações de hotplug de CPU exigem que você reinicie a máquina virtual.
====

== Ativar hotplug de CPU e Memória durante a criação da máquina virtual

. Na tela *Máquina Virtual: Criar*, especifique o número de núcleos de CPU e a memória a ser alocada.
+
image::vm/enable-cpu-and-memory-hotplug.png[Ativando hotplug de CPU e Memória]

. Selecione *Ativar hotplug de CPU e Memória*.
+
{harvester-product-name} preenche automaticamente os campos *Máximo de CPU* e *Máximo de Memória* com base na configuração `max-hotplug-ratio`. Esses valores representam a quantidade máxima de recursos de CPU e memória disponíveis para a máquina virtual durante a execução. Você pode alterar esses valores, se necessário.

. Complete o restante das etapas de criação da máquina virtual.

== Acionar hotplug de CPU e memória

. Após a criação da máquina virtual, você pode adicionar dinamicamente recursos de CPU e memória usando o botão *Editar CPU e Memória*.
+
image::vm/edit-cpu-and-memory.png[Aumentando recursos de CPU e memória]

. Clique em *Aplicar* para xref:virtual-machines/live-migration#_prerequisites[migrar a máquina virtual] para um nó com as quantidades de CPU e memória configuradas.

[NOTE]
====
A máquina virtual deve ser xref:virtual-machines/live-migration.adoc#_live_migratable_virtual_machines[migrável ao vivo].
====

== `max-hotplug-ratio` configuração

xref:installation-setup/config/settings.adoc#_max_hotplug_ratio[`max-hotplug-ratio`] é uma configuração global que determina a quantidade máxima padrão de recursos de CPU e memória disponíveis para uma máquina virtual em execução. Essa proporção é multiplicada pela quantidade de recursos de CPU e memória que você alocou ao criar a máquina virtual.

[sidebar]
Recurso máximo disponível em tempo de execução = (Recurso alocado durante a criação da máquina virtual) x (`max-hotplug-ratio` valor)

Você pode especificar um valor de `1` a `20`. O valor padrão é `4`.

Exemplo:

O `max-hotplug-ratio` está definido como `4`.

|===
| Resource | Quantidade Alocada | Quantidade Máxima Disponível

| Núcleos de CPU
| `1`
| `4`

| Memória (Gi)
| `2`
| `8`
|===
