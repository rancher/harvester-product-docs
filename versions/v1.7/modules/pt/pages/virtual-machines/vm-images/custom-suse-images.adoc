= Imagens de Máquinas Virtuais Personalizadas SUSE
:revdate: 2026-01-08
:page-revdate: {revdate}

A SUSE fornece https://www.suse.com/download/sles/[SUSE Linux Enterprise (SLE)] e https://get.opensuse.org/leap/[openSUSE Leap] imagens de máquinas virtuais adequadas para uso em {harvester-product-name}. Essas imagens são construídas no https://build.opensuse.org/[openSUSE Build Service] (OBS) usando a ferramenta de construção de imagens https://osinside.github.io/kiwi/[Kiwi], e podem ser usadas imediatamente após o download.

Na maioria dos casos, você pode usar as imagens qcow2 do _Minimal VM Cloud_ porque estas incluem a ferramenta cloud-init necessária para a configuração automática da máquina virtual. Outras variantes de imagem exigem que você faça login no console da máquina virtual e, em seguida, realize a configuração inicial.

[IMPORTANT]
====
As imagens _Minimal VM Cloud_ foram nomeadas _Minimal VM OpenStack Cloud_ em versões anteriores ao SLES 15 SP5 e openSUSE 15.5.
====

== Usando o openSUSE Build Service (OBS)

Você pode criar imagens personalizadas usando os https://build.opensuse.org/image_templates[modelos de imagem] do OBS da SUSE, que são configurações de imagem Kiwi pré-configuradas. Por exemplo, se você quiser usar outros pacotes com SLE 15 SP7, pode criar uma imagem usando o modelo _SLE 17 SP5 Minimal_. O OBS fornece uma interface para adicionar pacotes e constrói automaticamente a imagem, que você pode baixar e, em seguida, enviar para {harvester-product-name}. Para mais informações, consulte o https://openbuildservice.org/help/manuals/obs-user-guide/[Guia do Usuário do OBS].

=== 1. Crie uma imagem personalizada com base em um modelo existente.

. Vá para https://build.opensuse.org/image_templates. Você deve fazer login na sua conta openSUSE para acessar os recursos.
. Selecione o modelo que você deseja usar.
. Especifique um nome para a imagem e, em seguida, selecione *Criar appliance*.
+
image::advanced/custom-vm-01-select-template.png[]
+
O OBS constrói automaticamente a imagem. Por padrão, a interface mostra a aba *Visão Geral*, que contém informações como o número de pacotes incluídos e o status da construção.
+
image::advanced/custom-vm-02-image-overview.png[]

=== 2. Selecione arquivos de controle de imagem e adicione pacotes.

. Vá para a aba *Software*.
. Selecione os arquivos de controle de imagem que você deseja que o OBS construa.
+
[IMPORTANT]
====
Na maioria dos casos, você pode usar as imagens qcow2 do _Minimal VM Cloud_ porque estas incluem a ferramenta cloud-init necessária para a configuração automática da máquina virtual. Outras variantes de imagem exigem que você faça login no console da máquina virtual e, em seguida, realize a configuração inicial.
====
+
image::advanced/custom-vm-03-image-software.png[]

. (Opcional) Adicione e remova pacotes.
+
image::advanced/custom-vm-04-image-software-packages.png[]
+
image::advanced/custom-vm-05-image-software-add-package.png[]

=== 3. (Opcional) Mude para o modo _Visualizar Pacote_.

O modo _Visualizar Pacote_ oferece um controle mais granular sobre a configuração. Para mudar, clique no ícone _Visualizar Pacote_ na barra de navegação.

image::advanced/custom-vm-06-view-package.png[]

A seção *Arquivos-fonte* da aba *Visão Geral* mostra todos os arquivos que compõem seu template Kiwi. Você pode editar qualquer um dos arquivos selecionando o nome do arquivo correspondente.

image::advanced/custom-vm-07-view-package-details.png[]

=== 4. (Opcional) Edite o arquivo de configuração `Minimal.kiwi`.

Selecione o nome do arquivo para abrir o editor de texto. A seção `<packages type="image">` lista os pacotes a serem instalados. Você pode especificar pacotes adicionais para cada arquivo de controle de imagem.

Por padrão, o arquivo de controle de imagem _Cloud_ (`<package type="image" profiles="Cloud">`) instala o pacote _kernel-default-base_. No exemplo a seguir, esse pacote é substituído por _kernel-default_, que inclui módulos necessários para suporte a iSCSI.

image::advanced/custom-vm-08-edit-kiwi-config.png[]

=== 5. Aguarde o OBS terminar de construir a imagem.

Uma vez que o processo esteja concluído, a seção *Resultados da Construção* na aba *Visão Geral* mostra o status _sucedido_.

image::advanced/custom-vm-09-build-succeeded.png[]

A seção *Resultados da Construção* também contém um link para download da nova imagem.

image::advanced/custom-vm-10-binaries-link.png[]

image::advanced/custom-vm-11-binaries-download.png[]

=== 6. Ative a publicação para compartilhar a imagem.

Para permitir que o público baixe sua imagem personalizada, vá para a aba *Repositórios* do seu projeto OBS e ative a flag _Publicar_.

image::advanced/custom-vm-12-project-publish-repositories.png[]

Sua imagem é publicada em https://download.opensuse.org/ (sob `repositories/home:/YOUR_USER_NAME:/branches:/SUSE:/Templates:/Images:/`).

image::advanced/custom-vm-13-download.openuse.org.png[]

== Usando a Ferramenta de Linha de Comando Kiwi

Como uma alternativa ao openSUSE Build Service, você pode criar imagens localmente usando a ferramenta de linha de comando Kiwi. Para mais informações sobre a ferramenta, veja https://documentation.suse.com/appliance/kiwi-9/html/kiwi/index.html[Construindo Aparelhos de Sistema Linux com KIWI Next Generation (KIWI NG)].

Para criar imagens personalizadas, você deve primeiro baixar o arquivo `Minimal.kiwi`, e os scripts `config.sh` e `editbootinstall_rpi.sh` do projeto correspondente no OBS.
