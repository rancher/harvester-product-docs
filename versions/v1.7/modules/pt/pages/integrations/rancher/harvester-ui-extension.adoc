= Extensão da UI do Harvester
:revdate: 2025-12-30
:page-revdate: {revdate}

{rancher-short-name} As versões 2.10.0 e posteriores se integram com {harvester-product-name} usando a https://github.com/harvester/harvester-ui-extension[Extensão da UI do Harvester], que é construída sobre a biblioteca https://www.npmjs.com/package/@rancher/shell[@rancher/shell].

[NOTE]
====
A UI legada {harvester-product-name} continua a suportar v1.3.x e v1.4.x. Novos recursos em lançamentos posteriores são implementados na Extensão da UI do Harvester.
====

== Matriz de suporte

A Extensão da UI do Harvester é compatível com versões anteriores do {harvester-product-name}. Ela usa flags de recursos para exibir funcionalidades da UI que correspondem à versão do cluster {harvester-product-name}.

As versões do sistema operacional {harvester-product-name} e da Extensão da UI do Harvester se alinham a partir da v1.5.0. Para acessar novos recursos da UI, instale a versão mais recente da Extensão da UI do Harvester em sua instância {rancher-short-name}.

|===
| Versão da Extensão da UI do Harvester | Versões suportadas do {rancher-short-name}

| v1.0.x
| 2.10.1 e posteriores

| v1.5.x
| 2.11.0 e posteriores

| v1.6.x
| 2.12.0 e posteriores

| v1.7.x
| 2.13.0 e posteriores
|===

Instalar a extensão pela rede não é possível em ambientes air-gapped, portanto, você deve realizar a solução alternativa descrita em xref:../../installation-setup/airgap.adoc#_harvester_ui_extension_with_rancher_integration[Extensão da UI do Harvester com integração {rancher-short-name}].

== Instalação no {rancher-short-name} 2.10

=== Instalação automática

Você pode instalar a Extensão da UI do Harvester com um único clique.

. Na UI do {rancher-short-name}, vá para *Gerenciamento de Virtualização*.
+
image::rancher/auto-install-ui-extension.png[]

. Na aba *Clusters do Harvester*, clique em *Instalar*.
+
Aguarde um tempo para que a extensão seja instalada. A tela é atualizada automaticamente assim que a instalação é concluída.
+
[NOTE]
====
Não navegue para fora da tela durante o processo de instalação.
====

=== Instalação manual

Se a instalação automática falhar, você pode especificar a URL do repositório da Extensão da UI do Harvester e, em seguida, instalar a extensão.

image::rancher/ui-extension-install-failed.png[]

. Na UI {rancher-short-name}, vá para *local -> Apps -> Repositórios*, e então clique em *Criar*.
+
image::rancher/ui-extension-app-repository-setup.png[]

. Defina as seguintes configurações:
+
* Nome: Especifique um nome para o repositório.
* Destino: Selecione *Repositório Git contendo definições de gráfico Helm ou modelo de cluster*.
* URL do Repositório Git: Especifique *https://github.com/harvester/harvester-ui-extension*.
* Ramo Git: Especifique *gh-pages*.

. Clique em *Criar*.

. Vá para *Extensões -> Disponíveis*.
+
image::rancher/ui-extension-available-tab.png[]

. Localize a extensão chamada *Harvester*, e então clique em *Instalar*.

== Upgrades

=== Atualizações Automáticas

O botão *Atualizar* aparece sempre que uma nova versão da extensão está disponível.

. Na UI do {rancher-short-name}, vá para *Gerenciamento de Virtualização*.
+
image::rancher/ui-extension-update.png[]

. Clique em *Atualizar*.
+
Aguarde um tempo para que a extensão faça upgrade. A tela é atualizada automaticamente assim que o upgrade é concluído.

=== Upgrades Manuais

. Na UI {rancher-short-name}, vá para *local -> Apps -> Repositórios*.

. Localize o repositório com as seguintes configurações, e então selecione *⋮ -> Atualizar*.
+
* URL: *https://github.com/harvester/harvester-ui-extension*
* Ramo: *gh-pages*
+
image::upgrade/rancher-2.10.1-repository-page.png[]

. Vá para *Extensões -> Instaladas*.

. Localize a extensão chamada *Harvester* e clique em *Atualizar*.

. Selecione a nova versão e clique em *Atualizar*.

image::upgrade/update-harvester-ui-extension-modal.png[]

. Aguarde um tempo para que a extensão faça upgrade e, em seguida, atualize a tela.

== Limitações

=== {harvester-product-name} v1.2.x e versões anteriores não são suportadas

Quando você importar um cluster que está executando {harvester-product-name} v1.2.x ou versões anteriores, uma mensagem de erro aparecerá e o link do cluster será desativado.

O botão **Gerenciar** ainda está disponível na versão {rancher-short-name} 2.11.0, mas será desativado em uma versão futura. Para mais informações, consulte o problema https://github.com/harvester/harvester/issues/8054[#8054].

[CAUTION]
====
Evite usar o botão *Gerenciar* na interface {rancher-short-name} se seus clusters estiverem executando {harvester-product-name} v1.2.x ou versões anteriores. Gerenciar clusters usando este recurso pode resultar em comportamentos inesperados.
====
