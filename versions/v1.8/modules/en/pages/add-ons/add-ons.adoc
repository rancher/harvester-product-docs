= Add-ons
:revdate: 2025-12-30
:page-revdate: {revdate}

{harvester-product-name} provides optional features as add-ons to maintain a minimal installation footprint. This design allows you to enable or disable functionality based on your use case and requirements. You can customize the settings of each add-on, though the level of customization depends on the underlying software.

The following add-ons are available in this version:

* xref:add-ons/lvm-local-storage.adoc[csi-driver-lvm] (Experimental)
* xref:add-ons/kubeovn-operator.adoc[kubeovn-operator] (Experimental)
* xref:add-ons/nvidia-driver-toolkit.adoc[nvidia-driver-toolkit]
* xref:add-ons/pcidevices-controller.adoc[pcidevices-controller]
* xref:observability/logging.adoc[rancher-logging]
* xref:observability/monitoring.adoc[rancher-monitoring]
* xref:add-ons/rancher-vcluster.adoc[rancher-vcluster] (Experimental)
* xref:add-ons/harvester-seeder.adoc[seeder] (Experimental)
* xref:add-ons/vm-dhcp-controller.adoc[vm-dhcp-controller] (Experimental)
* xref:add-ons/vm-import-controller.adoc[vm-import-controller]

You can enable and disable add-ons on the *Add-ons* screen of the {harvester-product-name} UI.

|===
| Operation | Action | New State

| Enabling an add-on
| Select the add-on, and then select *⋮ -> Enable*.
| **DeploySuccessful**

| Disabling an add-on
| Select the add-on, and then select *⋮ -> Disable*.
| *Disabled*
|===

[NOTE]
====
{harvester-product-name} stores the configuration data of disabled add-ons, allowing for immediate reuse upon re-enabling.
====
